# 高城株式会社／株式会社デザインワークス様 AI構想提案（叩き台）

小早川和廉様（社長・部長）との面談用。建設・不動産＋設計のグループ（110-120名）向けに、
社長の構想④「**設計と個人情報をAIで移行したい**」を主題にした**ディスカッション用の叩き台**です。

> 方向性：④ ＝ **図面・書類の電子化＋個人情報の抽出・DB移行**（眠っているレガシー資産のデジタル化）として提案。
> この相手はAI成熟度が高く（公開API活用済み）、本件は業務効率化より**データ基盤プロジェクト**＝高単価・戦略案件。

## 構成（全12ページ）

1. 表紙　2. 本日のゴール（“正解を持ってくる”でなく一緒に決める）
3. 御社グループの理解　4. 社長の構想④を翻訳（いま→AIで移行→これから）
5. なぜ今か（眠った資産のリスク）　6. **全体像：図面・書類→AI抽出→DB**（4ステップ）
7. **個人情報・セキュリティの設計（肝）**　8. 進め方（PoC→本格移行→活用）
9. 広がり（同じ基盤で他業務も）　10. **★本日一緒に決めたいこと**（ヒアリング論点5つ）
11. 体制・効果イメージ　12. 次のアクション

## なぜこの構成か（営業設計の意図）

- **叩き台に徹する**：P2とP10で「完成提案ではなく、一緒に決める」と明言。社長に当事者として議論に乗ってもらう
- **個人情報を“最大の不安”でなく“こちらの強み”に**：P7で先回りして安全設計を提示。建設・不動産でAI活用が止まる最大の理由を潰す
- **小さく始める**：P8でPoCファースト。「いきなり全部」への警戒を解き、投資判断をしやすく
- **グループ横断の伸びしろ**：P9でデータ基盤の上に他アプリが載る絵を見せ、案件を大きく見せる

## 面談での使い方

1. P3で会社理解をすり合わせ（違えばその場で修正＝信頼獲得）
2. P4で④を「こう理解しました」と翻訳 → **認識合わせが最重要**
3. P6〜7で「やり方」と「安全性」を提示し、不安を先に潰す
4. **P10をメインに**ヒアリング。5論点（対象／現状／個人情報ルール／ゴール／体制）を埋める
5. P12で「まず1種類の書類でPoC」と次回約束を取る

### P10で必ず埋めたい5論点
- **対象**：まず何の書類・どの物件群から？（一番困っている／数が多い）
- **現状**：図面・顧客情報の今の形式は？（紙・PDF・既存システム・Excel）
- **個人情報**：社内ルール・NGライン・保管ポリシー、クラウド利用の可否
- **ゴール**：移行の先にやりたいこと（検索・再提案・営業・相続対応…の優先順位）
- **体制**：推進担当は誰か／高城・デザインワークスどちらから着手するか

## 個人情報の論点（面談前に頭出し）

建設・不動産は契約書・図面に個人情報が集中。AI活用の可否はここで決まります。資料P7の論点：
- 入力をモデル学習に使わない商用APIを選定（学習に流れない契約形態で運用）
- 送信前に氏名・住所等を検出してマスキング／最小限のみ処理
- アクセス権限・監査ログ／保管先（社内・閉域・国内クラウド）の選択
- 個人情報保護法・宅建業法への準拠、PoCで小さく検証

## 正直な注意書き

- ④の解釈は「図面・書類の電子化＋個人情報抽出・DB移行」と置いています。社長の真意とズレたらP4で修正を。
