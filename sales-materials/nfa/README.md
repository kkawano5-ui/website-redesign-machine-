# 株式会社エヌエフエー様 AI業務自動化 提案一式

大崎玄長社長との面談（来週）用。**「先方のオペレーションを想定 → 自動化提案 → 実際に動くデモ」** をワンセットにしています。

## 成果物

| ファイル | 用途 |
|---|---|
| `NFA様_AI業務自動化_提案資料.pptx` | 提案スライド（全14ページ・16:9）。面談のメイン資料 |
| `demo1_invoice-check.html` | **動くデモ①** 請求・給与ダブルチェック自動化（社長言及の業務そのもの） |
| `demo2_job-generator.html` | **動くデモ②** 求人原稿・スカウト・SNS 自動生成 |
| `build_nfa_pptx.py` | 提案PPTの生成スクリプト（再生成・編集用） |

デモはどちらも**単一HTMLファイル**。ダブルクリックでブラウザが開き、ネット接続不要でその場で動きます。

## 提案PPTの構成（全14ページ）

1. 表紙　2. 本日のゴール／アジェンダ　3. 御社の理解（PEO型・30名で1,000名）
4. 派遣ビジネスの構造的負荷　5. オペレーション全体像（5フロー想定）
6. **自動化の機会マップ**（効果×導入難易度の優先度マトリクス）
7. テーマ①請求チェック（本日デモ）　8. テーマ②求人原稿（本日デモ）
9. その他の自動化候補6つ　10. 想定効果・ROI　11. 90日ロードマップ
12. なぜ外注でなく研修（内製）か　13. デモのご案内　14. 次のアクション

## 面談の進め方（推奨フロー・約40分）

1. **つかみ（5分）** P3で「御社をこう理解しています」と返す → 社長の自動化志向に共感
2. **課題の合意（5分）** P4〜5で「30名で1,000名」の構造的負荷を一緒に確認
3. **優先度（5分）** P6のマップで「どこから」を可視化。社長の言葉（請求ダブルチェック）が右上＝最優先に来ている、と示す
4. **デモ①（8分）** `demo1` を開く。鈴木さんの**¥17,200差異**・伊藤さんの**マスタ欠落**が自動で🔴に出る瞬間を見せる →「これ、まさに今、人手でやられている確認ですよね」
5. **デモ②（5分）** `demo2` で条件を変えて原稿が一瞬で出るのを体感
6. **研修の価値（5分）** P12「外注で1個 vs 30名が作れる」で締める
7. **次の一手（5分）** P14。まず業務棚卸しワークショップから、と提案

## 面談で社長に聞きたいこと（提案を精緻化するため）
- 請求・給与のチェックは今、月に何件・何時間かけているか？
- 勤怠データはどんな形で上がってくるか（紙／Excel／システム）？
- 既に使っている基幹システム・販売管理ソフトは？（連携先の確認）
- 「かなり使っている社員」は何の業務にAIを使っているか？
- 30名のうち、推進役になれそうな人はいるか？

## 正直な注意書き（提案前にご確認を）
- **デモ①**：ルールベースの**本物の自動計算・突合**です（請求ダブルチェックはAI不要でこの方式が最適）。サンプルデータ入りで、御社の職種・単価に書き換えればその場で実演できます。
- **デモ②**：本デモは**テンプレート方式**の簡易生成です。**実装版はClaude API連携**で自然文を生成、と資料・画面にも明記済み。面談でも正直に伝えるのが信頼につながります。
- 会社情報・数値（1,000名規模、PEO等）は**公開情報＋ヒアリング内容ベース**。誤りがあれば差し替えてください。

