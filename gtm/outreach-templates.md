# アウトリーチ テンプレート（価値提供型・法令順守）

> 方針: スパムにしない。1社ずつ、実際に作ったデモを添えて「価値を先に渡す」。
> README方針どおり、当面は一括メール送信を行わず、質の高い個別提案を前提とする。

## コンプライアンス チェック（送信前に必読）

- [ ] 既存サイト等で**公開されている連絡先**にのみ送る。
- [ ] 特定電子メール法: 件名・本文に**送信者の氏名/会社名・連絡先・配信停止方法**を明記。
- [ ] 受信拒否の意思表示があれば**以後送らない**（記録を残す）。
- [ ] 既存サイトの文章・画像を流用しない（デモは独自制作）。
- [ ] 誇大・断定表現を避ける（業種別NG表現は site-spec の注意事項を参照）。

## 1. 初回コンタクト（メール / 問い合わせフォーム）

件名: 【{会社名}様】Webサイトのリニューアル案を作成しました（無料・確認用）

{ご担当者}様

突然のご連絡失礼いたします。{自社名}の{氏名}と申します。
地方の中小企業様向けに、Webサイトのリニューアル制作を行っております。

貴社サイトを拝見し、スマートフォンでの見やすさと問い合わせ導線について
改善できる点があると感じ、**確認用のデモページを試作**しました。

▼ デモ（{会社名}様向け・提案用）
{デモURL}

もしご興味があれば、15分ほどで改善ポイントと費用感をご説明します。
ご都合のよい候補日を2〜3いただけますでしょうか。

------------------------------
{自社名} {氏名}

配信停止をご希望の場合はこのメールにご返信ください。

## 2. フォローアップ（初回から3〜5営業日後・1回のみ）

件名: Re: 【{会社名}様】リニューアルデモの件

{ご担当者}様

先日お送りしたデモページの件、いかがでしたでしょうか。
{会社名}様の{強み/実績}が初見で伝わる構成を意識して作成しました。

▼ デモ
{デモURL}

「今は不要」「別担当へ」等でも構いませんので、一言いただければ幸いです。
（以後のご案内が不要な場合はその旨ご返信ください。）

## 3. 商談化したら（アジェンダ）

1. 現状サイトの課題（3点に絞る／site-spec の「既存サイトの課題」を活用）
2. デモで見せる改善後の体験（導線・スマホ・信頼性）
3. プラン提示（pricing.md のスタンダードを基準に）
4. 次アクション（契約 or 1週間の検討 + 修正1回）

## 運用メモ

- 1通ずつ「そのデモURL」を差し込むのが肝。`npm run build:site` / `build:all` で量産。
- 返信率を `送付数 / 返信 / 商談 / 成約` で記録し、`npm run model` の close 値を実績で更新。
- 反応の良い業種が見えたら、その業種の入力JSONを増やして集中する。
