# 料金設計・パッケージ（営業用）

デモを「見せて終わり」にせず契約につなげるための、価格と商品設計のたたき台。
数値は `npm run model` の前提と連動。実際の相場・原価に合わせて調整すること。

## 基本パッケージ

| プラン | 初期制作 | 月額保守 | 想定ページ数 | 主対象 |
|--------|---------:|---------:|------------:|--------|
| ライト | ¥180,000 | ¥8,000 | 1〜3 | 個人事業・小規模店舗 |
| スタンダード | ¥300,000 | ¥12,000 | 5〜7 | 地方中小企業（標準） |
| プロ | ¥500,000 | ¥18,000 | 8〜12 + 予約/問い合わせ強化 | 多店舗・採用強化したい企業 |

保守に含むもの: 文言・画像差し替え（月◯回）、軽微な追加、稼働監視、SSL/ドメイン管理。
別見積: 撮影、原稿ライティング、本格的なSEO記事制作、EC・予約システム連携。

## 価格を正当化する根拠（商談で使う）

- **意思決定までの導線短縮**: 問い合わせ動線を最短化し、機会損失を減らす。
- **スマホ最適化**: 検索流入の多くはモバイル。崩れたサイトは即離脱につながる。
- **信頼性**: 古いサイトは「営業しているか不安」と取られる。第一印象の改善。
- **更新性**: 保守契約で「更新できないまま放置」を防ぐ。

## 値引きより「条件」で守る

- 値引きを求められたら価格を下げず、**ページ数・撮影・保守期間**で調整する。
- 早期契約特典は「金額」ではなく「初月保守無料」「事例ページ1本サービス」等の非価格特典で。

## ユニットエコノミクス（目安）

- 1案件の粗利 = 初期制作 × (1 − 原価率) ＋ 保守の生涯価値（LTV）。
- LTV ≒ 月額 ÷ 月次解約率。例: ¥12,000 ÷ (20%/年 ≒ 月1.8%) ≒ 約65万円/契約。
- つまり**保守の継続が利益の本体**。制作は入口、保守がストック収益。

## 到達シナリオ

`npm run model -- --price 400000 --close 0.08 --demos 60 --retainer 18000`
→ 累計利益 ¥1億 到達まで約38ヶ月（前提次第）。
最も安く動かせる変数は **demos（送付デモ数）= このマシンの量産機能**。
