# SSK攻略・横展開インテリジェンス台帳
## ～SSKを起点にした静岡ドミナント・鈴与グループ・全国食品豪族展開のための情報設計書～

> 本台帳は、SSK（清水食品株式会社）担当者にそのまま渡す質問票ではない。
> 河野が商談・雑談・飲み・フォロー・提案・上司同席MTGの中で、自然に情報を拾うための「攻略インテリジェンス台帳」である。
> 質問は基本的に削っていない。複数回の接点で分割して埋めることを前提に、優先順位と場面別の聞き方だけを整理した。

## 第1章：この台帳の目的

### 1-1. なぜこの情報を取るのか

本台帳は、SSKを「1社の研修導入顧客」として終わらせず、当社が今後展開する以下のすべての営業活動の「起点」「証拠」「テンプレート」とするための情報設計書である。

- 静岡県内の中小・中堅企業へのドミナント展開
- 鈴与グループへの縦展開
- 全国の地方有力食品企業（豪族系）への横展開
- 地方企業向けAI研修営業の再現可能な型の確立
- SSKの取り組みを事例資産化し、認知・紹介・受注を生む装置に変える

### 1-2. SSKを「1社の顧客」と見ない理由

SSKは下記の特性を兼ね備える、極めて希少な「展開起点」である。

| 観点 | SSKの特性 |
|---|---|
| 地域代表性 | 清水を象徴する食品メーカー。はごろも・由比缶詰と並ぶ清水缶詰3社の一角 |
| グループ波及力 | 鈴与グループ清水経済圏に位置し、グループ波及の可能性 |
| 食品業界波及力 | 缶詰・レトルトという代表的食品カテゴリで全国の同業他社への再現性が高い |
| 既に顧客 | 担当者ファン化済み、信頼関係の土台が完成している |
| 事例化適性 | 創業家系・地元中堅という典型的「豪族系食品企業」プロファイル |

ゆえに本台帳は「SSKから何を売るか」だけでなく、「SSKから何を取り出すか」「SSKを起点に何を広げるか」に重点を置く。

### 1-3. 取り切るべき一次情報の質

机上で集まる情報（公開IR・業界レポート）では得られない、以下の一次情報を取り切ることが本台帳のミッションである。

1. 稟議が通った決定的な「言葉」と「数字」
2. 担当者の社内評価・キャリア野望・上司との力関係
3. 受講者が腹落ちした言葉、腹落ちしなかった言葉
4. 鈴与グループ内の意思決定構造とキーパーソン
5. SSK社長・経営層が反応した論点
6. 「他社（同業・グループ会社）に紹介できる／しやすい状態」の条件
7. 静岡県内・全国食品豪族へ転用可能な提案・稟議・成果フォーマット

### 1-4. 本台帳の位置付け

- これは河野の手元資料であり、SSK担当者に直接渡すものではない
- 1回の面談で全項目を聞かない。3〜6ヶ月で分割収集
- 一次情報と公開情報を分けて管理する
- 取った情報は「SSKのため」だけでなく「次の100社のため」に使う

## 第3章：情報収集の使い方

### 3-1. 大原則

1. **相手に質問票として渡さない**。台帳は河野の手元資料。
2. **1回の面談で全項目を聞かない**。3〜6ヶ月で分割収集。
3. **聞く→記録→分析→次の聞き方を設計、のサイクルを回す**。
4. **担当者本人だけが情報源ではない**。受講者、上司、役員、紹介者から三角測量する。
5. **公開情報と一次情報を区別する**。公開で取れるものは事前に埋め、面談時間は一次情報に集中。
6. **拾った情報は必ずその場で（または当日中に）台帳に書く**。記憶では失われる。

### 3-2. 拾った情報の保管

- 一次保管：SSK専用Notion／Scrapbox／スプレッドシート
- 鮮度管理：日付・取得経路・話者・発話の信頼度を必ず記録
- 機密区分：本人合意なし／合意あり／公開可、の3段階で管理
- 更新頻度：商談ごとに即日入力、月1で全項目レビュー
- 関連付け：担当者個別ノート／組織構造マップ／グループマップを連携

### 3-3. 情報源の三角測量

| 取りたい情報 | 一次情報源 | 二次情報源 | 検証情報源 |
|---|---|---|---|
| 稟議の中身 | 担当者本人 | 担当者上司 | 受講者 |
| 担当者の社内評価 | 受講者・他部署 | 上司 | 役員クラス |
| 鈴与グループ動向 | 担当者 | 鈴与HD公開IR | 清水銀行・商工会議所 |
| 競合の動き | 受講者 | 担当者 | 業界紙・SNS |
| 成果定量数値 | 担当者 | 受講者 | システムログ |

## 第5章：場面別の聞き方・拾い方

### 5-1. 商談で聞くこと（フォーマル）

商談はゴールが明確な場。質問は「成果確認→次施策→横展開示唆」の順で組み立てる。

| カテゴリ | 質問例 | 拾うべき項目 |
|---|---|---|
| 成果確認 | 「研修後、具体的にどんな変化が部内で起きていますか？」 | D, E, F |
| 言語化支援 | 「もし役員報告でひとこと言うなら、何と言いますか？」 | G, R |
| 次施策 | 「次に解決したい課題は何ですか？」 | H, M |
| 横展開示唆 | 「同じ課題を持っている部署はありそうですか？」 | H |
| グループ示唆 | 「グループ会社でも同じような話、出てますか？」 | I |
| 競合確認 | 「他社のAI研修も検討されたことありますか？」 | L, M, R |

### 5-2. 飲み・雑談で拾うこと（インフォーマル）

「教えてください」ではなく「○○さんって、こういう感じですよね」と仮説をぶつけて返ってくる反応で拾う。

| カテゴリ | 拾い方 | 拾うべき項目 |
|---|---|---|
| 担当者本人 | 趣味・家族・出身・学生時代の話 | B |
| 上司との関係 | 「上司の○○部長ってどんな方ですか？」 | B |
| キャリア観 | 「将来は○○みたいなことやりたいですか？」 | B, K |
| 社内の派閥 | 「グループ内で清水と本社って、関係どうですか？」 | I, L |
| 競合の影 | 「他にも色んなAI研修の営業来てます？」 | L, M |
| 不満・愚痴 | 受講者・上司・役員への本音 | B, E, L |
| 個人野望 | 「もしSSKを辞めたら何やりたいですか？」 | B, K, L |

### 5-3. 上司・役員同席時に観察すること（観察）

質問する場ではない。観察に徹する。

- 上司は担当者を「立てている」か「軽く見ている」か
- 上司の発言に「DX」「AI」「生成AI」「人材育成」のどれが出るか
- 役員クラスの認知レベル（名前まで知っているか／取り組み内容まで知っているか）
- 役員が反応する話題（コスト削減／競争力／採用／人手不足／取引先要請）
- 上司・役員が「ここから先は」と話題を変えるトピック（タブー領域）
- 担当者が同席上司の発言時に見せる顔（賛同／違和感／不満）
- 上司・役員の意思決定の主導性（即決型／持ち帰り型／合議型）
- 上司が「鈴与本社」「鈴与HD」「グループ」をどう呼ぶか（距離感の表れ）

### 5-4. 提案後の反応から読むこと（リアクション分析）

| 提案後の反応 | 解釈 | 次のアクション |
|---|---|---|
| 即答で「やりましょう」 | 担当者の権限内、または事前根回し済み | スピード重視で具体化 |
| 「持ち帰って相談します」＋具体日付 | 上司決裁が必要、現実的に進む | 持ち帰り資料の支援 |
| 「持ち帰ります」＋曖昧 | 担当者は前向きだが社内ハードルあり | 障壁の特定、別人接触検討 |
| 「予算が…」 | 金額より優先順位の問題 | 補助金・分割・小額PoCを提示 |
| 「他にもいくつか聞いてます」 | 競合検討中、または断り文句 | 競合の正体特定 |
| 沈黙・スルー | 担当者の社内事情で動けない | 担当者の立場を察し、ペース調整 |
| 「面白い提案ですね」だけで具体化なし | 関心はあるが社内ニーズ未醸成 | 関連他部署を巻き込む |

### 5-5. 成果報告・追加提案時に確認すること

成果報告会は「次の予算」を取りに行く最大の機会。

- 役員クラスが報告に同席するか
- 報告書のフォーマット指定があるか
- 想定質問への事前準備の必要性
- 報告後の次フェーズ稟議のスケジュール
- 「成果」として認められる定量基準
- 競合・社内のネガティブ勢力の存在
- 報告会後に担当者から共有してもらえる「役員の反応」
- 報告会の場での無料セミナー告知の可否
- 報告会後の懇親会の有無（インフォーマル情報取得機会）

### 5-6. グループ展開・紹介依頼時に確認すること

紹介依頼は「担当者にリスクを負わせる」行為。慎重に。

- 紹介先との担当者の関係性（同期／先輩／後輩／面識のみ）
- 紹介の温度感（強推薦／顔つなぎ／名前出しのみ）
- 紹介後に担当者へどう報告するか
- 紹介がうまくいかなかった場合の担当者へのフォロー
- 紹介依頼のタイミング（成果が見えてから／報告会の直後／次年度予算の前）
- 紹介がうまく回った場合の担当者へのリターン（実績認定／社内告知／謝礼）
- 紹介先での当社の振る舞いルール（担当者の顔を立てる／勝手に進めない）
- 紹介後の進捗共有の頻度（毎週／月1）

## 第6章：A〜Vの詳細項目

### A. 導入・受講経緯（優先度:A／場面:商談、上司同席）

導入トリガー
- 誰が最初に「AI研修を入れよう」と言い出したのか
- そのきっかけ・トリガーは何だったか（経営指示／個人課題感／競合事例／業界トレンド／補助金／取引先要請／鈴与本社からの宿題）
- いつから検討を始めたか、検討期間はどれくらいだったか
- 検討段階で見た他社サービスはあったか、何と何と何を比較したか
- なぜ最終的に当社を選んだか、決定要因の上位3つは何か

意思決定プロセス
- 稟議は誰がどのレイヤーまで通したか、最終決裁者は誰か
- どの部署のどの費目から予算を出したか（人事教育費／DX推進費／経営企画費／補助金活用）
- 予算規模は社内的に「お試し」「中規模」「大型」のどれに分類されているか
- もし当社を選ばなかった場合、SSKはどう動いた可能性があるか
- 「導入しなかった場合の機会損失」をどう社内で説明したか

時期的文脈
- 研修導入と社内イベント（中期計画策定／DX宣言／決算／組織改編）の時期的関係
- 鈴与グループ側の動き（HD方針発表、グループ会議等）との連動性
- 食品業界・地域経済のトレンドとの連動性

### B. 担当者の人物像（優先度:B／場面:雑談、飲み、長期1on1）

属性
- 氏名・所属・役職・年齢・社歴
- 新卒入社か中途か、前職・学歴
- 評価制度上、何をKPIにされているか
- 評価面談のタイミング（賞与査定・昇格審査の月）

社内ポジション
- 直属上司は誰か、関係性（信頼度・距離感）
- 役員クラスへの直接報告ラインはあるか
- 社内タイプ（ファシリテーター型／コネクター型／専門家型／改革者型）
- 部署内ポジション（次期課長候補／既に管理職／部長代行）
- 社内的な味方・後ろ盾は誰か
- 社内的な抵抗勢力・敵対関係は誰か

野望と不満
- このプロジェクトに賭けているもの（評価／キャリア／自己実現）
- 「やりたいけど社内政治でできていない」こと
- 本人の悩み・困りごと
- 上司の悩み（担当者から見て）
- キャリア野望（管理職志向／専門家志向／独立志向）

個人情報（雑談蓄積）
- 家族構成・趣味・地元・出身校・所属コミュニティ／学会
- 出したい「成果報告」のフォーマット
- 社外発信（SNS／登壇／執筆）への前向き度
- 異動可能性・退職可能性（業界平均と本人志向から）

### C. 研修内容と実施の棚卸し（優先度:A／場面:商談、成果報告会）

実施事実
- 受講者の部署・役職・人数・任意か必須か
- 受講者の年齢層・PCリテラシー・職種内訳
- 研修の構成（コマ割／単元／演習比率）
- 期間・スケジュール（連続／分散／実施月）
- 形態（対面／オンライン／ハイブリッド／録画／ライブ）
- 使用ツール（ChatGPT／Claude／Gemini／Copilot／自社業務連携）

事前期待値
- 受講者の事前期待値（事前アンケート）
- 担当者の事前期待値
- 上司・役員の事前期待値

運用実態
- 演習で実業務データを使えたか
- 宿題・課題の提出率
- 修了テスト・修了証の有無
- 講師⇄受講者のコミュニケーションチャネル（Slack／Teams）
- 研修後のフォローアップ設計の運用度

講師評価
- 講師の評価（受講者アンケート）
- 講師個人の固有名と紐づけた評判

### D. 成果・変化・インパクト（優先度:A／場面:商談、成果報告会）

定量
- 受講前後スキル変化（テストスコア）
- 業務時間削減（hours/week, hours/month, 円換算）
- 業務処理件数の増加
- アウトプット品質スコア
- 受講後ChatGPT/Claude継続利用率（DAU/WAU/MAU）
- 1人あたり週次プロンプト数

## 第9章：3ヶ月検証戦略への接続

3ヶ月でSSKから取り切るべき情報と、それに基づく判断軸。

| 情報項目 | YES時のアクション | NO時のアクション |
|---|---|---|

| 鈴与グループへの紹介意欲 | 鈴与本社（人事・DX）接触を開始 | グループ展開は保留、SSK内深耕優先 |
| SSK内2部署目展開合意 | 横展開設計に着手・成果報告フォーマット量産 | 担当者の社内ポジションを再評価 |

| 役員クラスの認知獲得 | 役員直接接触の段取り | 担当者経由の発信を強化 |
| 業界誌取材合意 | 全国食品豪族営業の打ち手に転用 | 当社サイト事例化を優先 |

### 静岡ドミナント着手の判断基準
- SSKの事例ロゴが提案書に使える状態か
- SSKでの成果数値が「具体的」に語れる状態か
- SSK担当者が静岡銀行・清水銀行・商工会議所と接続できるか
- SSK担当者の協力で静岡県内2社への接点が作れるか

### 鈴与グループ展開判断基準
- SSK担当者が鈴与本社の人事・DX部門のキーパーソンを知っているか
- グループ横断会議体への持ち込み可能性があるか
- SSK役員クラスの推薦を得られるか
- 鈴与HDのDX戦略・教育方針との整合性が取れるか

### 全国食品豪族展開判断基準
- SSK事例が「業界共通課題」として語れる構造になっているか
- 業界誌・業界団体経由の露出経路が作れるか
- 共同登壇・寄稿の協力が得られるか
- 稟議テンプレート・成果報告テンプレートが他社転用可能な汎用性を持つか

## 第10章：撤退・継続判断に使う情報

### 継続強化のサイン
- 担当者が役員から名指しで褒められた
- 受講者から自発的に「もっとやりたい」声が出た
- 他部署から「うちもやりたい」打診が来た
- 鈴与グループ会社から問い合わせが入った
- 静岡県内同業から「SSKがやってる研修って何？」と聞かれた
- 業界メディアからの取材打診が入った
- 担当者が自発的に社外発信を始めた
- 役員クラスから直接の問い合わせ・打診が入った

### 撤退・リソース転換のサイン
- 担当者が異動・退職になり、後任との関係が築けない
- 上司が交代し、新上司がAI研修に否定的
- 競合が低価格で受注を奪い始め、防衛が困難
- 成果が定量化できず、報告会で評価されなかった
- 鈴与グループ内で別ベンダーが包括契約を取った
- 受講者からの満足度が継続的に下落
- 担当者が紹介に消極的・他部署展開を渋る

### 担当者の異動・退職時の対応
- 異動先での再アプローチ（担当者本人を異動先のチャンピオン化）
- 後任への引き継ぎMTG同席（担当者協力下で）
- 過去事例のアーカイブを担当者に手渡し（思い出資料化）
- 退職後もコミュニティで関係維持（OB会・業界イベント）
- 担当者異動の事前察知（評価面談時期・本人ヒント）

### 競合参入時の対応
- 競合の差別化ポイントを担当者から正確に把握
- 当社の独自価値の再言語化
- 役員クラスへの直接訴求パスの確立
- 「変更コスト」「現場混乱」「実績のないリスク」の可視化
- 補助金・追加メニュー・カスタム性での差別化

### 撤退基準（厳格）
以下の3つ以上が同時発生した場合、リソース転換を検討する。

1. 担当者が交代し3ヶ月以内に後任との関係構築失敗
2. 役員クラスが当社サービスを「不要」と発言
3. 競合が包括契約で勝ち、当社が部分契約に縮小
4. 成果報告で「研修の効果なし」と社内結論
5. 鈴与HD方針で別ベンダー指定
