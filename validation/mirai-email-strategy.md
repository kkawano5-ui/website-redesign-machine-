# みらい編集｜メール営業 計測＆スケール戦略（500件 → 数字 → 本気のリスト獲得）

> 親方針：**まず500件を“超細かく”計測し、開封率→クリック率→CVRまで数字を出す。CVRが成立したら、リスト獲得を本気でやる。**
> 文面 → `validation/mirai-email-copy.md` ／ トラッカー → `validation/mirai-leads-tracker.csv`
> このフェーズの riskiest assumption：**「中小企業オーナーに、コールドメールで“相談”まで引き込めるか。引き込めるなら1件いくらか」**

---

## 0. 全体像（このファネルの数字を全部取る）

```
送信(Sent) → 到達(Delivered) → 開封(Open) → クリック(Click) → LP着地
        → 反応(返信 or 相談申込 = CV) → 商談(通話/訪問) → 受注(Won)
```

| 指標 | 定義 | 取り方 | 中小BtoBコールドの目安※ |
|---|---|---|---|
| 到達率 | 届いた / 送信 | ツールのbounce数 | ≥ 95%（低いとリスト品質 or 迷惑判定） |
| **開封率** | 開封 / 到達 | トラッキングピクセル | 30〜50%（後述の注意あり） |
| **クリック率** | クリック / 到達 | リンクトラッキング | 2〜5% |
| 返信率 | 返信 / 到達 | 受信箱を手動 or ツール | 1〜5% |
| **CVR** | 相談申込 / クリック | ★LPフォーム必須（後述） | 5〜15%（クリック→申込） |
| アポ率 | 通話/訪問 / 送信 | 台帳手動 | 0.5〜2% |
| 受注率 | 受注 / 商談 | 台帳手動 | 商談の10〜30% |

※目安は一般的なレンジ。**自社の実数が出たらそれが唯一の基準**。最初の500件は「自社ベンチマークを作る」ためのもの。

## 1. 計測の作り込み方（指標ごと）

### 開封率（トラッキングピクセル）
- 配信ツールが本文に1px画像を仕込み、表示されたら開封としてカウント。**どのツールでも標準機能**。
- ⚠️ **Apple Mail / iCloudのプライバシー保護**で開封が“水増し”される（実際は開いてなくてもカウント）。→ 開封率は**方向性の参考**にとどめ、判断の主役は**クリック率と返信率**に置く。
- A/Bは**件名**で回す（開封に最も効く変数）。

### クリック率（リンクトラッキング＋UTM）
- 本文リンクをツールが計測用URLに自動置換 → クリックを集計。
- 加えて**UTMを必ず付ける**（文面に記載済み）：
  - `utm_campaign=mirai_w1`（週/バッチ単位で変える）
  - `utm_content=onestop / cost / meo`（**どの文面が効いたか**が分かる）
- これでツール側クリック数とLP側流入（GA4/Plausible）が**二重で取れる**。

### 返信率
- 「この返信に〇〇と返してください」をCTAにしているので、返信は最重要シグナル。
- ツールに返信検知があれば自動、無ければ受信箱を見て台帳に記録。

### CVR（クリック→申込）★ここだけ未着手＝要フォーム
- 厳密なCVR（クリック→申込）には、**みらい編集LPに計測付き“無料相談フォーム”が必要**。現状LPは電話/mailtoのみで、フォーム送信イベントが取れない。
- **今回は文面・戦略優先のため、フォームは未追加**（ユーザー方針）。それまでの間の暫定CVR：
  - 暫定CVR = **（返信 + 電話着信 + mailtoクリック）/ クリック** で代用。
  - 電話は「このメール見た」と一言もらう or 専用番号で識別。
- **本気のリスト獲得フェーズに入る前に、フォーム追加を必ず実施**（→ コンシェルジュLPと同方式：ネイティブフォーム＋UTM隠しフィールド＋Formspree。実装は別タスクで即可能）。

## 2. 送信ツール比較（※「相談したい」とのことなので推奨つき）

| ツール | 開封/クリック計測 | コールド向き | 月コスト目安 | 備考 |
|---|---|---|---|---|
| **Instantly / Lemlist**（コールド特化）| ◎ 開封/クリック/返信を自動追跡 | ◎ ウォームアップ・1対1送信で到達性高 | $30〜50 | **500件量産の本命**。複数差出しでスケール |
| **Gmail＋Mailmeteor / Streak**（差込配信）| ○ 開封ピクセル＋クリック | △ 1日の送信上限あり・大量は不向き | ¥0〜数百 | **最初の数十〜100件のテストに最適・無料で始められる** |
| Brevo / Mailchimp（メルマガ系ESP）| ◎ 標準で開封/クリック | ✕ **コールド/購入リストは規約違反**で凍結リスク | ¥0〜 | オプトイン済みリスト向け。今回のコールドには不適 |
| 国産（blastmail / Benchmark等）| ◎ | △ 同上、オプトイン前提 | ¥数千〜 | 日本語サポート◎だがコールド非推奨 |

### 推奨（2段ロケット）
1. **検証フェーズ（最初の50〜100件）＝ Gmail＋Mailmeteor で¥0スタート。**
   - 既存 `kawano@tv-hospital.com`（または営業用に別ドメイン）から、差込配信＋開封/クリック計測。手早く文面A/Bの初速を見る。
2. **量産フェーズ（残り〜500件＆その先）＝ Instantly か Lemlist。**
   - ウォームアップで迷惑メール落ちを防ぎつつ、開封/クリック/返信を自動集計。500件→数千件のスケールに耐える。
- **Brevo/Mailchimpはコールドでは使わない**（凍結リスク）。オプトインのリストが育ったら配信に使う、という住み分け。
- ⚠️ どのツールでも**送信ドメインの SPF / DKIM / DMARC 設定**を先に済ませる（到達率と開封率の土台。未設定だと全部が迷惑メール行きで数字が死ぬ）。

## 3. 500件フェーズの回し方（“超細かく”見る）

> 一気に500通送らない。**100件×5バッチ**に割り、各バッチで件名/文面をA/Bして学習しながら回す。

| バッチ | 件数 | 何を試す | 見る数字 |
|---|---|---|---|
| B1 | 100 | 件名A vs B（開封の差） | 開封率の勝ち件名 |
| B2 | 100 | 勝ち件名で 文面① vs ②（クリックの差） | クリック率・返信率 |
| B3 | 100 | 勝ち文面で 業種セグメント別（飲食 vs 美容 vs 工務店…）| セグメント別の反応 |
| B4 | 100 | 勝ちパターンでフォロー有無の差 | フォローの上乗せ効果 |
| B5 | 100 | 確定版で再現性チェック | 数字が安定するか |

- 各バッチ後に `mirai-leads-tracker.csv` を集計 → 次バッチに反映。
- **1要素ずつ変える**（件名と文面を同時に変えると、どっちが効いたか分からなくなる）。

### 判定ゲート（500件終了時 → 「本気のリスト獲得」へ進むか）
- **GO（本気で量産）**：以下がそろう
  - クリック率 **≥ 3%** が安定 ＆ 返信率 **≥ 2%**
  - 相談/商談（通話 or 訪問）が **500件で ≥ 5件**
  - うち**受注 or 強い前向き ≥ 1件**（＝価格を伏せても相談まで来る経路が成立）
- **GOしたらやること**：①LPに計測フォーム追加（CVRを正式計測）②リスト獲得を本気で（後述）③Instantly等で送信量を10倍に。
- **KILL/見直し**：丁寧にやってクリック<1%・返信ほぼ0 → 文面 or ターゲット or チャネル（メール以外＝架電/フォーム営業/ポスティング/MEO広告）を見直す。

## 4. 「本気のリスト獲得」フェーズ（CVRが出たら）

> CVRと1件あたりコストが見えてから初めて、お金と手間をかけてリストを増やす。順番厳守。

- **リスト源**：iタウンページ/Googleマップ/業界団体名簿/エキテン等から、対象業種×地域で**公開事業用アドレス**を収集（特定電子メール法の例外範囲を守る）。
- **獲得コストを計算**：1件のリスト獲得コスト＋送信コスト → CVRで割って**CAC（顧客獲得単価）**を出す。受注1件のLTV（電気代→MEO→SNS→採用…の積み上げ）と比較。
- **自動化**：MA/コールド送信ツールでセグメント配信。受注者は除外、未反応は再ナーチャ。
- **横展開**：勝ち文面を架電トーク・フォーム営業・DM・MEO/SNS広告にも転用（チャネル横断で同じ訴求を当てる）。
