# 集客検証プラン｜ヘリテージ・ウェディング（destination wedding Japan）

> 親方針：既存の `validation/acquisition-validation-plan.md` と同じ規律（**安い順に・タイムボックスで・単位経済で go/kill を引く**）を、ウェディング縦軸に適用する。
>
> 送り先 LP：`wedding/index.html`（concierge と同じ**UTM付き問い合わせフォーム計測**を実装済み）。
>
> 検証する riskiest assumption：**「日系ルーツの英語圏HNWに、想定経路で到達でき、CPAが¥100k前後（=ペイする水準）で有効リード→成約まで運べる」**

---

## 0. 結論サマリ（なぜこの縦軸が筋がいいか）

- **広告で売れる商材**。一生に一度・オンラインで計画・ビジュアル命。コンシェルジュ案の鶏卵問題（実績ゼロに客が回らない）が無い。
- **プランナーの50件超の実績＝最初から最強のクリエイティブと信用**。広告素材・推薦文・写真が手元にある。
- **堀は供給側**：英語ネイティブ対応 × 実績のあるプランナーが希少。模倣困難。

## 1. ユニットエコノミクス（5/5/5モデル）

| 項目 | 数字 |
|---|---|
| 平均挙式単価 | **¥8,000,000** |
| 粗利（売上の15%） | **¥1,200,000 / 組** |
| ├ プランナー 5% | ¥400,000 / 組 |
| ├ 友達 5% | ¥400,000 / 組 |
| └ Raha Kenya 5% | **¥400,000 / 組** |
| 目標 | **月5組**（GMV ¥40M/月・¥480M/年） |
| Raha Kenya 年間取り分（5組×12ヶ月） | **¥24M/年** |

**CPAの効き**：目標 CPA ¥100k は Raha 取り分(¥400k)の25%・**粗利全体(¥1.2M)の8.3%・GMVの1.25%**。
→ 成立すれば異次元に効率がいい。**CPAが本当に¥100kで立つか**だけが生命線。それをこのプランで先に潰す。

### ファネルの目安（CPAの内訳）
```
CPA = 有効リード単価(CPQL) ÷ 成約率
推定CAC = CPQL ÷ 成約率
```
| 成約率（有効リード→成約） | CPA ¥100k に必要な CPQL |
|---|---|
| 10% | ≤ ¥10,000 |
| 15% | ≤ ¥15,000 |
| 20% | ≤ ¥20,000 |

> 注：たとえCACが¥200〜300kでも Raha取り分¥400k内でペイする。**¥100kは保守目標**。だから判定はCAC≤¥150kでGO、≤¥400kはグレー（経路ミックス前提）、>¥400kでKILL（有料単独は不経済）と置く。

## 2. ⚠️ 最重要の制約：集客とプランナー供給はセットで縛る

- 友達1人 ＝ **月1.5件が上限**。**月5件には同等プランナーが3〜4人**要る。
- このプランナーこそが堀（希少資源）。**集客だけ5件に増やすと品質崩壊か待ち行列**になる。
- → ランプ設計：**月1.5〜2件でCPAを証明 → 2人目採用 → 段階的に5件へ**。友達は「稼ぐプランナー」から徐々に「リードプランナー＝採用・教育・品質管理」へ。
- **キャッシュ注意**：着金は挙式の8〜18ヶ月前。CPAは先払い・売上は後着。デポジットで運転資金を埋める。

## 5. go/kill判定（単位経済で下す）

| 結果（推定CPA = CPQL ÷ 成約率） | 判定 |
|---|---|
| 推定CPA ≤ **¥150k** が見える | **GO**：有料直販が成立。月1.5〜2件で本稼働→採用で階段拡大 |
| ¥150k〜¥400k | グレー：有料単独は薄い。**紹介・コミュニティ・Pinterest と混ぜて** CPAを薄める前提で再設計 |
| > **¥400k** or 全く刺さらない | **KILL**：有料検索は今は不経済。**紹介・コミュニティ・SEO・提携**に資源集中 |

- **有効問い合わせの定義**：実在の人物／日系ルーツ or 日本挙式の明確な意向／日程・人数・関心が埋まっている（スパム・冷やかし除外）。
