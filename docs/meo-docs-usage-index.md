# MEO資料の使い方インデックス（これ1枚で全部わかる）

> 作った資料は全部、**「営業 → 受注 → オンボーディング → 日々の運用 → レポート/更新」という業務の流れ**のどこかで使う道具です。どの場面でどれを開くかを、この1枚に集約しました。

---

## 0. 全体像（業務フロー × 使う資料）

```
【営業】────────【受注/契約】───【オンボーディング】──【日々の運用】──【レポート/更新】
提案書ジェネレーター   契約書修正案      オンボーディング台本   エスカレ・ルール   運用プレイブック
サービス詳細         サービス詳細                          運用台帳          運用台帳
                                                        提案書ジェネレーター（次の営業へ）
```

---

## 1. 資料ごとの「いつ・何のため」早見表

| 資料 | いつ開く | 何のため |
|---|---|---|
| **提案書ジェネレーター**（GitHub `proposal-generator/`） | 営業前 | Googleマップのリンク→提案書PPTXを自動生成 |
| **サービス詳細（8ライン）** | 営業・教育・線引き確認 | 何を売れるか／できない事の正典 |
| **運用モデル・意思決定ログ** | 迷ったとき・新メンバー・別セッション | なぜこう決めたかの全記録 |
| **契約書 修正案** | 受注・契約時 | 竹の公開代行・承認方式・自動化同意を契約に反映 |
| **オンボーディング面談 台本** | 受注直後の面談 | 権限取得＋運用方針合意＋自動化同意を45分で |
| **エスカレーション・ルール** | 日々の運用・実装 | 自動で流す/人に上げるの判断基準 |
| **運用台帳（Sheet）** | 毎週 | 全店のステータス・順位・未返信・更新日を一元管理 |
| **運用プレイブック** | 受注前チェック・日々 | 属人化防止・スケール・ポリシー・解約防止の型 |
| **提案書リバイス・ブリーフ** | 提案書を直すとき | デッキ修正の指示＋貼り込み文言（デザイン変更時に再利用） |

## 2. フェーズ別の使い方

### フェーズ0：準備・教育（受注ゼロの今やること）
- **サービス詳細**を読み込み、「できること／できないこと」を体に入れる（営業で言い切れるように）。
- **運用プレイブック**の§6「受注前チェックリスト」をYesにしていく。
- **運用モデル**は、チーム全員が最初に読む"全体地図"。

### フェーズ1：営業
1. 見込み店舗の**Googleマップのリンク**を用意。
2. `proposal-generator` で **提案書PPTXを自動生成**（会社名・エリア・業種テーマ・現状診断が入った状態）。
3. 商談で提示。トークは既存の営業トークシート（Notion/Drive）と併用。
4. 「順位保証はしない・受け皿づくり」の語り口は**サービス詳細§0**の通りに。

### フェーズ2：受注・契約
- **契約書 修正案**の新旧対照どおりに契約書を用意（竹＝投稿掲載代行／承認は"運用方針＋例外"／自動化同意条項）。
- ⚠️ 契約書の最終版は**弁護士レビュー**を通す。

### フェーズ3：オンボーディング（受注直後・本人が実施）
- **オンボーディング面談 台本**を開いて45分の面談。ここで：
  - GBP権限をもらう（Manager以上／顧客はPrimary Owner保持）
  - 運用の"憲法"（投稿テーマ方針・口コミ姿勢）を合意
  - **自動化の同意**（★4-5自動返信／承認済みテーマ自動公開／ネガは都度相談）
- 面談で埋めたシートを**運用台帳**に1行追加（業種＝薬機法ルールパックが効く）。ベースライン（開始時順位・インサイト）を記録。

### フェーズ4：日々の運用
- **エスカレーション・ルール**が判断基準：★4-5クリーンは自動、★1-3やトリガー語・薬機法アウトは人へ。これで「45承認/日→5〜7件/日」。
- **運用台帳**を**毎週1回**全店ぶん更新（ステータス列＝どこがヤバいか一目）。
- **運用プレイブック**§3-2の週次ルーティンに沿って回す。

### フェーズ5：レポート・更新（解約防止）
- 月次レポート（自動生成→送信）。**運用プレイブック§5**の「4要素＋解約防止の習慣」に沿う。
- **運用台帳**の更新日30日前アラートで更新面談→次の一手を提案。

## 3. 「困ったら」逆引き

| こんなとき | 開く資料 |
|---|---|
| この施策やっていい？（薬機法・口コミ依頼） | エスカレーション・ルール／サービス詳細§5 |
| 顧客に「これできる？」と聞かれた | サービス詳細（8ライン・できる/できない） |
| なんでこの設計にしたっけ？ | 運用モデル・意思決定ログ |
| 提案書のデザイン/内容を変えたい | 提案書リバイス・ブリーフ |
| 新しい店を受注した | オンボーディング台本→運用台帳 |
| 受注が増えて回らない | 運用プレイブック§4（スケール）／エスカレ（自動化率UP） |
