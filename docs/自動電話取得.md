# 電話番号を「毎日◯件だけ・勝ち筋の順に」自動取得する

Google Places の無料枠（**10,000呼び出し/月/SKU ≒ 1日約333件**）に自動で収まるよう、
**優先度スコアの高い会社から毎日N件ずつ**ドリップ取得する仕組みです。都度バッチを手で回す必要はありません。

- **あてずっぽうにしない**：業界ティア×サイト有無×口コミ×エリアでスコア化し、上位から叩く（下記）
- **実測データも同時取得**：電話と同じ1回のAPIで website/★評価/口コミ実数/営業状態も取る（**追加課金ゼロ**＝同一Enterprise SKU）
- 取得済みは自動スキップ → 毎日走らせるだけで、勝ち筋業種→新規追加ぶん、と自動で埋まります
- 1日の件数は自由に変更可（既定 **300件/日** = 月9,000件で無料枠内）
- 出力は `data/outputs/phones.csv`（A〜H列は据え置き＝シートのVLOOKUPは不変。I列以降に優先度・実測を追記）

## 叩く順番（優先度）と実測データ

**「適切な業界の適切なお客様」から**埋めるため、スコア順に取得します（`scripts/fetch-phones.mjs` 上部で編集可）：

| 軸 | ルール | 加点 |
|---|---|---|
| 業界 | 美容/治療院/エステ/歯科（予約集客型・MEO直結） | +88〜100 |
| 業界 | その他＝飲食など | +55 |
| 業界 | 士業（税理士/司法書士等・Web制作の的だがMEO弱め） | +40 |
| 業界 | 公共・非商用（官公庁/学校/モール等）＝**叩かない** | 除外 |
| 客層 | サイト無し（Web提案の的） | +30 |
| 客層 | 口コミ1〜50（気にしてるが手が回ってない＝ホット） | +20 |
| 客層 | 口コミ200超（繁盛店・ネット強者＝難易度高） | −10 |

→ 例：**美容×サイト無し×口コミ数件 = 100+30+20 = スコア150** が最上位。同スコアはエリアで固まる（架電しやすい）。

出力の追加列：`優先度`（I）／`website実測`（J）／`★評価`（K）／`口コミ実数`（L）／`営業状態`（M。CLOSED=閉業は架電除外の判断に）。
シートで **優先度列で降順ソート**すれば、そのまま「今日叩くリスト」になります。

## 初回セットアップ（Macで1回だけ）

**1) APIキーを Keychain に安全に保存**（平文でファイルに置かないため）

```bash
security add-generic-password -a "$USER" -s miraihen-places-key -w 'AIza...ここにキー...'
```

> `.env` に `GOOGLE_MAPS_API_KEY=AIza...` を書いてもOK（その場合はこの手順は不要）。

**2) 毎日の自動実行を登録**（既定：毎日9:00に300件）

```bash
bash scripts/install-phones-schedule.sh
# 例) 朝8時に200件にしたい:
# bash scripts/install-phones-schedule.sh --hour 8 --daily 200
```

これで完了。あとはMacが起きていれば毎日ひとりでに走ります（スリープ中は次の起床時にまとめて実行）。

## よく使うコマンド

```bash
# いますぐ1回試す（登録済みのジョブを即実行）
launchctl kickstart -k gui/$(id -u)/com.miraihen.fetch-phones

# 手動で1回だけドリップ実行（件数はDAILYで指定可）
DAILY=100 npm run fetch:phones:daily

# 既存2032社の実測データ(website/★/口コミ実数)を一気に埋めたい（無料枠内なので一括でもOK）
DAILY=2032 npm run fetch:phones:daily

# 実測エンリッチを止めて「電話番号だけ」に戻す
REENRICH=0 npm run fetch:phones:daily

# 実行ログを見る
tail -f data/logs/fetch-phones.log

# 件数や時刻を変えたい → もう一度インストールを実行（上書き登録）
bash scripts/install-phones-schedule.sh --hour 7 --daily 250

# 自動実行をやめる
bash scripts/install-phones-schedule.sh --remove
```

## 仕組みメモ

- `scripts/fetch-phones.mjs --daily N` … 未処理のうち**優先度順で先頭N件**だけ取得して終了（`--no-priority`で入力順）
- `--reenrich` … 取得済みでも実測データ(website/★/口コミ実数)が無い行を取り直す（既定ON。追加課金ゼロ）
- `scripts/fetch-phones-daily.sh` … Keychainからキーを取り出し、`--daily`＋`--reenrich`でログ付き実行するラッパー
- `scripts/install-phones-schedule.sh` … 上記を launchd に「毎日◯時」で登録/解除

**優先度の重み・除外業種・エリアの扱いは `scripts/fetch-phones.mjs` 上部（SERVICE_WEIGHT / SHIGYO / EXCLUDE_GENRE / customerSignal）で編集可。**
編集すれば翌日のドリップ順に即反映されます。

対象の会社リストは `data/companies/crm-source.csv`。
**範囲を広げる**ときは、このCSVに会社（`place_id`列が必須）を追記すれば、翌日以降のドリップで優先度順に自動で拾われます。
初回は既存2032社の実測エンリッチ（勝ち筋業種＝美容/治療院等から）が優先的に進み、その後に新規追加ぶんへ移ります。
