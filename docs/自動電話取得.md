# 電話番号を「毎日◯件だけ・勝ち筋の順に」自動取得する

Google Places の無料枠（**10,000呼び出し/月/SKU ≒ 1日約333件**）に自動で収まるよう、
**優先度スコアの高い会社から毎日N件ずつ**ドリップ取得する仕組みです。都度バッチを手で回す必要はありません。

- **あてずっぽうにしない**：業種スコア（業界ティア×サイト有無×口コミ）で並べ、上位から叩く（下記）
- **実測データも同時取得**：電話と同じ1回のAPIで website/★評価/口コミ実数/営業状態も取る（**追加課金ゼロ**＝同一Enterprise SKU）
- 取得済みは自動スキップ → 毎日走らせるだけで、勝ち筋業種→新規追加ぶん、と自動で埋まります
- 1日の件数は自由に変更可（既定 **300件/日** = 月9,300件で無料枠内）

やり方は2通り。**方法A（シート直結・推奨）**が最少工数です。

---

## 叩く順番（優先度スコア）

**「適切な業界の適切なお客様」から**埋めるため、スコア順に取得します（業種スコアのみ。エリア重みなし）：

| 軸 | ルール | 加点 |
|---|---|---|
| 業界 | 美容/治療院/エステ/歯科（予約集客型・MEO直結） | +88〜100 |
| 業界 | その他＝飲食など | +55 |
| 業界 | 士業（税理士/司法書士等・Web制作の的だがMEO弱め） | +40 |
| 業界 | 公共・非商用（官公庁/学校/モール等）＝**叩かない** | 除外(0) |
| 客層 | サイト無し（Web提案の的） | +30 |
| 客層 | 口コミ1〜50（気にしてるが手が回ってない＝ホット） | +20 |
| 客層 | 口コミ200超（繁盛店・ネット強者＝難易度高） | −10 |
| 実測 | 営業状態が「閉業」 | 0（叩かない） |

→ 例：**美容×サイト無し×口コミ数件 = 100+30+20 = スコア150** が最上位。
実測が取れた行は**実測ベースで自動再計算**（例: 実は本物のサイトがあった美容 → 150→120 に降格）。

## 方法A（推奨）: シート直結 — スプレッドシートが自分で毎日取得する

`gas/phone-drip.gs` を貼るだけ。**Mac不要・CSV不要・貼り付け作業ゼロ。**
毎朝7時台に CRMシートへ直接書き込みます（電話・優先度・website実測・★評価・口コミ実数・営業状態・実測取得日）。

**セットアップ（3分・1回だけ）**

1. スプレッドシートの「拡張機能 > Apps Script」（crm-automation と同じプロジェクト）で
   「＋ > スクリプト」→ 名前 `phone-drip` → `gas/phone-drip.gs` の全文を貼り付けて保存
2. 左の歯車「プロジェクトの設定」> **スクリプト プロパティ** > プロパティを追加
   `PLACES_API_KEY` = `AIza...`（Placesのキー）
   ※キーの「**アプリケーションの制限」は「なし」**にする（IP制限だとGASから叩けない）。
   代わりに「APIの制限」で **Places API (New) だけ**に絞ると安全。
3. エディタ上部で **`setupPhoneDrip` を選んで ▶実行**（初回は承認ダイアログ）
   → 明日から毎日 朝7時台に自動実行

**動作確認**: `testPhoneDrip` を▶実行（5件だけ取得）→ CRMの右端に実測列が埋まればOK。
**今すぐ1回ぶん回す**: `dripFetchPhones` を▶実行（最大300件）。

**安全ルール（既存データを壊さない）**
- 電話列は **空/「有」/「-」/「なし」のセルだけ**埋める。既にある番号は絶対に上書きしない
- 追加6列（優先度〜実測取得日）が無ければ**右端に自動作成**。既存列の位置・数式・VLOOKUPは不変
- 失敗した行は「実測取得日」を付けない＝**翌日自動リトライ**
- 二重起動はロックで防止。プログラムからの書込は onEdit トリガー（集計/商談転記）を発火させない

**範囲を広げるとき**: 新エリアの会社（place_id付き）の行をCRMに貼るだけ。
翌朝から優先度順に自動で電話番号が埋まっていきます。

**件数/時刻/重みの変更**: `phone-drip.gs` 上部の `DRIP.dailyLimit` / `DRIP.triggerHour` / `serviceWeight` 等を編集
→ 時刻を変えたら `setupPhoneDrip` をもう一度▶実行（登録し直し）。

## 方法B（予備）: Mac の launchd で毎日実行 → CSV出力

シートに直接書かず `data/outputs/phones.csv` に貯める版。GASが使えない時のバックアップ。

**1) APIキーを Keychain に保存**（平文でファイルに置かない）

```bash
security add-generic-password -a "$USER" -s miraihen-places-key -w 'AIza...ここにキー...'
```

**2) 毎日の自動実行を登録**（既定：毎日9:00に300件）

```bash
bash scripts/install-phones-schedule.sh
# 例) 朝8時に200件: bash scripts/install-phones-schedule.sh --hour 8 --daily 200
```

**よく使うコマンド**

```bash
launchctl kickstart -k gui/$(id -u)/com.miraihen.fetch-phones  # いますぐ1回
DAILY=100 npm run fetch:phones:daily                            # 手動で1回（件数指定）
tail -f data/logs/fetch-phones.log                              # ログ
bash scripts/install-phones-schedule.sh --remove                # 自動実行をやめる
```

CSVの列: A〜H（place_id/電話/…/status）は従来どおり、I〜N に 優先度/website実測/★評価/口コミ実数/営業状態/enriched。
`--no-priority` で入力順、`--reenrich` で実測未取得の取得済み行も取り直し（ラッパーは既定ON）。

## 仕組みメモ

- **方法A**: `gas/phone-drip.gs` … CRMシートの place_id を直接読み、優先度順に Places API (New) を叩いて同じ行に書き戻す。
  対象＝「place_id あり・実測取得日なし・スコア>0」の行。毎日のトリガーは `setupPhoneDrip` が管理。
- **方法B**: `scripts/fetch-phones.mjs --daily N [--reenrich]` ＋ `scripts/fetch-phones-daily.sh` ＋ `scripts/install-phones-schedule.sh`。
  入力は `data/companies/crm-source.csv`（place_id列必須）。
- 優先度の重み・除外業種は **AとBで同一**（片方を変えたらもう片方も合わせる）:
  GAS側 `DRIP.serviceWeight / shigyo / excludeGenre`、Node側 `SERVICE_WEIGHT / SHIGYO / EXCLUDE_GENRE`。
- 課金メモ: 電話番号は Place Details の Enterprise SKU。website/★/口コミ実数/営業状態は**同じ呼び出しに同梱で追加課金なし**。
  無料枠は SKUごとに 10,000/月 → 300件/日なら枠内。
