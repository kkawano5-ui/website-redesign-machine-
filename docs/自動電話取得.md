# 電話番号を「毎日◯件だけ」自動取得する

Google Places の無料枠（**10,000呼び出し/月/SKU ≒ 1日約333件**）に自動で収まるよう、
**未取得の会社を毎日N件ずつ**ドリップ取得する仕組みです。都度バッチを手で回す必要はありません。

- 取得済みは自動スキップ → 毎日走らせるだけで、新しく追加した会社ぶんが少しずつ埋まります
- 1日の件数は自由に変更可（既定 **300件/日** = 月9,000件で無料枠内）
- 出力は `data/outputs/phones.csv`（従来どおり。スプレッドシートへは VLOOKUP で反映）

---

## 初回セットアップ（Macで1回だけ）

**1) APIキーを Keychain に安全に保存**（平文でファイルに置かないため）

```bash
security add-generic-password -a "$USER" -s miraihen-places-key -w 'AIza...ここにキー...'
```

> `.env` に `GOOGLE_MAPS_API_KEY=AIza...` を書いてもOK（その場合はこの手順は不要）。

**2) 毎日の自動実行を登録**（既定：毎日9:00に300件）

```bash
bash scripts/install-phones-schedule.sh
# 例) 朝8時に200件にしたい:
# bash scripts/install-phones-schedule.sh --hour 8 --daily 200
```

これで完了。あとはMacが起きていれば毎日ひとりでに走ります（スリープ中は次の起床時にまとめて実行）。

---

## よく使うコマンド

```bash
# いますぐ1回試す（登録済みのジョブを即実行）
launchctl kickstart -k gui/$(id -u)/com.miraihen.fetch-phones

# 手動で1回だけドリップ実行（件数はDAILYで指定可）
DAILY=100 npm run fetch:phones:daily

# 実行ログを見る
tail -f data/logs/fetch-phones.log

# 件数や時刻を変えたい → もう一度インストールを実行（上書き登録）
bash scripts/install-phones-schedule.sh --hour 7 --daily 250

# 自動実行をやめる
bash scripts/install-phones-schedule.sh --remove
```

---

## 仕組みメモ

- `scripts/fetch-phones.mjs --daily N` … 未取得のうち先頭N件だけ取得して終了
- `scripts/fetch-phones-daily.sh` … Keychainからキーを取り出してログ付きで実行するラッパー
- `scripts/install-phones-schedule.sh` … 上記を launchd に「毎日◯時」で登録/解除

対象の会社リストは `data/companies/crm-source.csv`。
**範囲を広げる**ときは、このCSVに会社（`place_id`列が必須）を追記すれば、翌日以降のドリップで自動的に拾われます。
