# エヌエフエー向け 実践ワーク題材集（具体例）

> カリキュラムの各テーマを「現状の手作業 → 入力データ例 → AIにやらせること → 出力イメージ → 効果」まで具体化したもの。
> 研修ワークショップの題材、および提案時に「分かってる感」を出すための素材として使う。
> 数値・データはすべて**仮の例**（実データはヒアリング後に差し替え）。

## 主力職種の仮置き
公開情報（大田区＝京浜・羽田の物流圏／BPOで「物流・ロジ・梱包・出荷・検品・軽作業・倉庫」「コールセンター」を明記／商談での前払い・短期・大量稼働）から、
**物流・軽作業（倉庫/梱包/出荷/検品）＋コールセンター** を稼働ボリュームの中心と仮置きして題材化する。
（介護・保育・事務・宿泊なども扱うため、最終的な題材職種はヒアリングで確定。）

## 各題材がどのコース・回に対応するか（早見）
- 配置転換マッチング → コースB第3回（差し替え）／コースA第3回
- 前払いの毎朝処理 → コースC第4回／コースA第5回
- 請求＋ダブルチェック → コースC第3回／コースA第4回
- 契約・法定帳票の自動生成 → コースC第5回／コースD第3回
- 労使協定方式の賃金突合 → コースC第5回
- 研修センターのAI化・教育訓練 → コースA/D第6回
- 請負（CC/物流）の現場・品質管理 → コースC第5回（請負分）／コースD

## 題材① 配置転換マッチング（PEOの生命線・稼働率に直結）

**現状の手作業**：倉庫Aの繁忙契約が今週末で終了する軽作業スタッフ8名。待機させず次の就業先へ即配置したいが、コーディネーターが頭の中とExcelで「誰をどこへ」を突き合わせている。

**入力データ例**
- 就業終了予定者リスト：`氏名／保有スキル（フォーク資格・検品経験等）／希望エリア／時給／就業可能曜日・時間帯／個人抵触日`
- 空き求人リスト：`派遣先／職種／必要人数／時給／勤務地／開始日／必要スキル`

**AIにやらせること**
- スキル・エリア・時給・開始日の適合度をスコアリングし、各スタッフに合う求人を上位提示。
- 個人抵触日・雇用安定措置の対象かどうかの制約も加味（3年到達見込み者は別扱い）。
- マッチ理由とミスマッチ理由を一言で添える。

**出力イメージ**
- スタッフ別「次の配置先 候補トップ3（理由つき）」
- 「充足できない求人」「行き先が見つからない待機リスク者」のアラート

**効果**：待機日数の短縮＝稼働率アップ（PEOの利益の源泉）。コーディネーターの突合時間を大幅削減。

## 題材② 前払い（日払い）の毎朝処理

**現状の手作業**：毎朝、スタッフから前払い申請メールが届く → 勤怠と照合 → 規定内か確認 → 振込データを手作業で作成。「朝までに必ず」の時間的プレッシャーが重い。

**入力データ例**
- 申請メール：`氏名／勤務日／申請額`（文面はバラバラ）
- 勤怠実績、前払い規定（例：稼働実績の○％まで等）

**AIにやらせること**
- メール本文を解析して申請内容を構造化 → 勤怠実績と突合 → 規定上限内かチェック → 振込用データ（CSV）を生成 → 不一致・上限超過は例外フラグ。

**出力イメージ**
- 振込用CSV（銀行フォーマット）／要確認リスト／スタッフへの返信文面ドラフト
- ※銀行への実送金は当面「データ生成まで自動・実行は人が承認」が安全。

**効果**：朝の処理時間を圧縮、人手の照合ミスを排除。

## 題材③ 請求の自動生成＋AIダブルチェック

**現状の手作業**：月末、派遣先ごとに勤怠を集計して請求書を作成。派遣先ごとに単価・時間外割増・締めサイクルが違い、転記・計算ミスが起きやすい。利益率が薄いのでミスが利益を直撃。

**入力データ例**
- 契約マスタ：`派遣先／職種／単価／時間外割増率／締めサイクル（月末／木曜締め等）`
- 勤怠データ（派遣先承認済み）

**AIにやらせること**
- 勤怠×契約マスタで請求額を算出 → 契約条件（単価・割増・締め）と整合しているかをAIが再チェック → 差異を検出。

**出力イメージ**
- 派遣先別の請求書（PDF/HTML）＋「契約条件と合わない箇所」の差異レポート

**効果**：請求ミスの撲滅（＝利益保全）、発行リードタイム短縮。AIが人の作業をダブルチェックする運用。

## 題材④ 契約・法定帳票の自動生成（個別契約→就業条件明示書→派遣元管理台帳）

**現状の手作業**：新規就業ごとに、個別契約書・就業条件明示書・派遣元管理台帳を作成。**同じ情報を3つの帳票に転記**していて手間と転記ミスの温床。

**入力データ例**
- 受注/ヒアリング情報：`派遣先／業務内容／期間／就業場所／指揮命令者／賃金／時間外の有無／抵触日`

**AIにやらせること**
- 1つの入力から3帳票を一括ドラフト生成 → 法定必須項目の抜け漏れチェック → 抵触日の整合チェック。

**出力イメージ**
- 個別契約書／就業条件明示書／派遣元管理台帳のドラフト一式＋チェックリスト（不足項目の指摘）

**効果**：転記作業をゼロに近づけ、コンプラ抜け漏れを防止。契約が正本なので下流（請求・給与）の精度も上がる。

## 題材⑤ 労使協定方式の「一般賃金」突合（毎年の重作業）

**現状の手作業**：毎年8月頃の局長通達（職種別の賃金水準・地域指数）の更新を受け、賃金テーブルを作り直し、全スタッフの賃金が水準を下回っていないか突合。職種が多いほど膨大。

**入力データ例**
- 局長通達データ：`職種別基準値／能力・経験調整指数／地域指数（東京）`
- 自社スタッフの賃金台帳：`氏名／職種／現在時給`

**AIにやらせること**
- 一般賃金 ＝ 基準値 × 能力経験調整指数 × 地域指数 を職種別に算出 → 自社時給と比較 → 下回り者を抽出 → 是正に必要な額を試算。

**出力イメージ**
- 職種別の賃金テーブル／水準を下回るスタッフ一覧／是正額の試算

**効果**：年次の地獄作業を圧縮、法令割れ（最低基準割れ）リスクを回避。

## 題材⑥ 研修センターのAI化＋キャリアアップ教育訓練

**現状の手作業**：多職種（物流・CC・事務など）向けの研修コンテンツ作成と、全員に年8時間以上（許可要件）の教育訓練の受講管理。

**入力データ例**
- 職種別スキル要件、スタッフの受講履歴

**AIにやらせること**
- 職種別の研修教材・チェックテストを生成 → 受講履歴から「年8時間未達」者を抽出 → 労働局提出用の教育訓練計画のドラフト作成。

**出力イメージ**
- 研修教材＋習熟度テスト／未受講アラート／教育訓練計画ドラフト
- ★ここに **AIスキルアカデミーの教材** を組み込めば、「研修を受けること自体が御社の法定教育訓練（年8時間）の達成にもなる」という提案につながる（要件適合は要確認）。

**効果**：教育の内製を効率化し、許可要件を確実に充足。PEO（習熟人材育成）の中核を強化。

## 題材⑦ 請負（コールセンター/物流）の現場・品質管理

**現状の手作業**：請負現場のシフト作成、稼働実績の集計、品質（応対品質・誤出荷率等）の管理。請負は「業務の完成」責任なので派遣とは管理の質が違う。

**入力データ例**
- 必要人数・時間帯、スタッフの稼働可能枠、過去の実績（CCなら応対件数・後処理時間／物流なら処理量・誤出荷）

**AIにやらせること**
- シフトのたたき台を自動作成 → 実績の集計・異常検知 → 偽装請負リスク（現場の指揮命令者が誰か）の観点チェック。

**出力イメージ**
- シフト案／実績サマリー＋異常アラート／請負コンプラのチェック結果

**効果**：現場管理の工数削減、請負特有のリスク（偽装請負）の早期検知。
