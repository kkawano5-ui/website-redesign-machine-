## 2. 論点（補助金審査・遂行の観点）

### 論点A：債務超過（純資産マイナス）
- 3期連続でマイナス、かつ拡大（-65k→-45k→-108k）。
- 即失格ではないが、**「事業の継続性・遂行能力」で減点**されやすい。
- → 役員借入の**資本振替（DES）や増資**で純資産を改善できると印象が大きく変わる。

### 論点B：直近期の売上ゼロ＝事業実態が弱い
- 補助金は「**事業を営んでいる**小規模事業者」が対象。直近売上ゼロは「実態がない」と見られるリスク。
- → 本格稼働後、**直近期にきちんと売上を計上**し、役員報酬を設定して「動いている会社」にするのが先。

### 論点C：🚨 最大の壁＝資金繰り（後払い・自己負担）
- 補助金は**後払い**。例えば補助200万を得るには、**先に250〜400万円を自費で支払う**必要。
- 現状：現預金 約12万円・債務超過。**自力では補助事業の支払いができない。**
- → **つなぎ資金の確保が大前提**（下記）。ここが解決しないと、どんなに良い計画でも実行不能。

### 論点D：補助率アップ（賃上げ特例）は現状非現実的
- 役員報酬ゼロ・従業員ゼロでは「給与総額の増加」要件を満たしにくい。
- → 4/5などの高補助率は当面狙わない（無理しない方針と一致）。

### 論点E：提出書類が古い
- いただいたのは第2〜4期（2021〜2023）。**補助金は直近期の決算書**が必要。
- 現在は **第6期（2024/10–2025/9）まで確定しているはず**。→ **最新の決算書（第5期・第6期）を要提出。**

## 3. 進め方（方針：最初からIT補助金200万を狙う）

> ユーザー意向：**最初からIT補助金200万を狙う**。事業実態（売上612万）はクリアしたので方向性は妥当。
> ただし**資金繰りと債務超過の手当てを“申請前”に並行で進める**ことが絶対条件。

### STEP 1（最優先・申請前）：増資で「債務超過の解消」＋「自己資金の確保」を同時に
- 現状：純資産 -79万・現預金4万。このままでは①審査で不利②支払い不能。
- 打ち手：**社長からの増資（現金）**、または**役員借入（80万）の資本振替（DES）**。
  - 例）現金増資300万＋役員借入80万をDES → 純資産が大きくプラスに転換、手元資金も厚くなる。
- 効果：**債務超過を解消（論点A解決）**＋**補助事業の支払原資を確保（論点C緩和）**。これが一番効く一手。

### STEP 2（並行）：日本政策金融公庫のつなぎ／設備融資
- 補助金は後払い。**公庫融資とセット**が王道（公庫も補助金採択を歓迎）。
- 補助対象経費の支払い→入金までのギャップを融資で埋める。

### STEP 3：gBizIDプライム・SECURITY ACTION 取得 → 交付申請
- 申請の中身（事業計画・労働生産性・経費明細）は `01`〜`05` をベースに実数化。
- **交付決定後に発注・支払い**（順番厳守）。

> ※ 資金繰り・増資の目処が立たないまま交付申請に突っ込むのは厳禁。採択後に支払えず辞退・返還となると信用を損なう。
