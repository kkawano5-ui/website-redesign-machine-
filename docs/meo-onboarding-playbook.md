# MEO オンボーディング面談 台本（45分・本人が実施）

> **役割**：受注直後の面談で「①関係構築 ②ヒアリング ③現状診断の共有 ④運用方針の合意 ⑤自動化の同意＋GBP権限取得 ⑥素材回収」を45分で終えるための台本。
> **なぜ重要か**：ここで決めた"運用の憲法"（投稿テーマ方針・口コミの向き合い方・自動化の同意）が、以後の**毎日の個別承認を不要にする**。承認は「毎日1件ずつ」ではなく「オンボーディングで一度まとめて」行う。
> **回収項目はデータ直結**：この面談で埋めるオンボーディングシート（§4）が、そのまま `data/meo/stores/{id}.json`（store-config）と運用台帳に転記される。

---

## 1. 事前準備（面談前・15分）

- [ ] **初期診断を済ませる**：対象GBPの現状（カテゴリ／説明文／写真／口コミ数・返信状況／NAP整合性／競合3社）をチェックし、メモ化。面談で画面共有して見せる。
- [ ] オンボーディングシート（§4）を開いておく。
- [ ] 権限付与の手順書（§5）を画面共有用に準備。
- [ ] 対策KWの仮説（地域＋業種＋ニーズ）を用意。

---

## 2. 面談アジェンダ（45分・分刻み）

| 時間 | セクション | ゴール |
|---|---|---|
| 0–5分 | アイスブレイク＋ゴール共有 | 場をほぐす／今日決めることを握る |
| 5–15分 | ヒアリング | 強み・ターゲット・NG表現を引き出す |
| 15–22分 | 現状診断の共有 | "受け皿の穴"を見せて納得感をつくる |
| 22–30分 | 運用方針の合意 | 投稿テーマ方針・口コミの向き合い方を確定（＝憲法） |
| 30–38分 | **自動化の同意＋権限付与** | ★4-5自動返信／承認済みテーマ自動公開／ネガは都度相談 の合意＋GBP権限受領 |
| 38–43分 | 素材回収＋スケジュール | 写真・事例・営業時間を回収／初期整備の日程 |
| 43–45分 | クロージング | 効果は3ヶ月／レポートの見方／次回連絡 |

## 3. セクション別 台本

### 3-1. アイスブレイク＋ゴール共有（0–5分）
**目的**：緊張をほぐし、今日のゴールを握る。

> 「本日はお時間ありがとうございます。今日は45分で、**お店の強みをうかがって、Googleマップでの見せ方の方針を一緒に決めるところ**まで進められればと思っています。難しい設定はこちらで全部やるので、○○さんは"どんなお店にしたいか"だけ、リラックスして話してください。」

- [ ] 今日のゴール（方針合意＋権限共有）を口頭で握る

### 3-2. ヒアリング（5–15分）
**目的**：投稿・口コミ返信の"素"になる情報を引き出す。ここで話してもらった言葉が、そのまま投稿トーンになる。

**トーク例**：
> 「まず、お店の**一番の強み**や、他店と違うところを教えてください。」
> 「どんなお客様に来てほしいですか？（年代・地域・悩み）」
> 「逆に、**"これは言われたくない／使ってほしくない表現"**ってありますか？」
> 「過去に、言い方でヒヤッとした・クレームになったことは？」

**回収するもの（→ store-config）**：
- [ ] 店の強み・こだわり（3つ）
- [ ] ターゲット顧客（年代・地域・ニーズ）
- [ ] ブランドトーン（丁寧／親しみ／高級感 など）
- [ ] **NG表現・避けたいこと**（→ ポリシーチェックの店舗別追加ワードにも反映）
- [ ] 業種の確定（→ `industry` フィールド＝薬機法等のルールパックが自動で効く）

### 3-3. 現状診断の共有（15–22分）
**目的**：「比較されたときに負けている点」を見せ、価値を体感してもらう。

> 「事前に今のGoogleマップを拝見しました。基本情報は整っているんですが、**比較されたときに"あと一歩"なのがこの3点**です。〔写真／口コミ返信／投稿の更新〕…。ここを継続的に整えると、**同じ検索で並んだときに選ばれやすく**なります。」

- [ ] 現状の穴を3点、具体で見せる（競合との差も）
- [ ] ※ここでも**順位保証はしない**（「選ばれやすい状態を整える」と言う）

### 3-4. 運用方針の合意（22–30分）＝"憲法"づくり
**目的**：毎月の投稿・口コミ対応の"方針"をここで一度決める。以後はこの範囲で自動運用。

**投稿テーマの方針（→ `approvedThemes`）**：
> 「毎月の投稿は、だいたい**この4テーマを回す**形にしようと思います：〔①サービス紹介 ②お客様の悩み別 ③季節・キャンペーン ④豆知識〕。この方向でいいですか？"これは避けたい"テーマがあれば今のうちに。」

**口コミへの向き合い方**：
> 「口コミは、**良い評価には感謝のお返事、厳しい評価には誠実にお詫びと改善**、という姿勢で統一します。○○さんとして"この点は特に丁寧に返したい"ことはありますか？」

- [ ] 承認済み投稿テーマを4つ確定（＝これが自動生成・自動公開の許可範囲）
- [ ] 口コミ返信の基本姿勢を合意

### 3-5. 自動化の同意＋権限付与（30–38分）★最重要
**目的**：自動運用の同意を取る（Googleの第三者ポリシー上、代理店が運用するには顧客の明示同意が必要）＋GBP権限を受領する。

**同意を取るトーク（そのまま使える）**：
> 「運用の進め方を1つだけ決めさせてください。日々の投稿と口コミのお返事は、**AIで下ごしらえして、私が薬機法や表現をチェックしてから出す**形にしています。速さと正確さの両立のためです。そのうえで——
>
> ① **高評価（★4〜5）へのお礼のお返事**は、こちらで整えて**自動でお返し**してよいですか？（内容はいつでも見られますし、直せます）
> ② **今日決めた投稿テーマの範囲**は、毎回ご確認いただかなくても、こちらで作って**掲載まで進めて**よいですか？
> ③ 一方で、**低評価・クレーム・デリケートな内容**は、**必ず一度ご相談してから**対応します。勝手には出しません。
>
> この進め方でよければ、**GBPの管理権限だけ共有**をお願いできますか。」

- [ ] ①★4-5自動返信の同意
- [ ] ②承認済みテーマの自動公開の同意
- [ ] ③ネガティブは都度相談（エスカレーション）を合意
- [ ] ④ **GBP権限の付与**（§5の手順でその場 or 後日）
- [ ] ⑤ この同意は契約書にも反映される旨を一言（→ `docs/meo-operating-model.md` §4-2）

> 💡 ①②③がまさに §エスカレーション・ルールの「自動 vs 人」の線引き。**ここで顧客と握る＝毎日の承認を先に済ませている**。

### 3-6. 素材回収＋スケジュール（38–43分）
**目的**：初期整備に必要なものを集め、日程を握る。

> 「初期整備に、**お店の写真（外観・内観・スタッフ・事例）**と、**営業時間・定休日・電話番号**をいただけると助かります。写真は今あるものでOK、後からでも大丈夫です。」

- [ ] 写真の提供方法・期限（原則毎月25日ルールも案内）
- [ ] 営業時間・定休日・電話・住所（NAP）
- [ ] 事例・お知らせしたいトピック
- [ ] 初期整備の開始日を握る

### 3-7. クロージング（43–45分）
> 「ありがとうございます。**効果は積み上げ型なので、最低3ヶ月**は一緒に育てさせてください。毎月、**表示回数・電話・ルート検索などの数値をレポート**でお送りします。順位そのものはGoogleの仕組み次第なので保証はしませんが、数値は正直に全部お見せします。次回レポートは〔日付〕にお送りしますね。」

- [ ] 効果は3ヶ月（積み上げ型）を握る
- [ ] レポートの見方・頻度を案内
- [ ] 次回レポート日を伝える

## 4. オンボーディングシート（回収項目＝データモデル直結）

面談中に埋め、当日中に `data/meo/stores/{id}.json` と運用台帳へ転記。

```
【基本】店舗名 / 正式名称 / 業種(industry) / 住所 / 電話 / 営業時間 / 定休日 / GBP URL
【方針】対策KW(5〜10) / ブランドトーン / 強み3点 / ターゲット / NG表現
【承認済みテーマ】theme1〜4（＝自動生成・自動公開の許可範囲）
【自動化の同意】★4-5自動返信: 可/不可 ／ 承認済みテーマ自動公開: 可/不可 ／ ネガ都度相談: 合意
【エスカレーション】連絡先・連絡方法（LINE/メール/電話）・返答期待スピード
【権限】GBP権限付与: 済/未（Owner or Manager）
【素材】写真: 有/後日 ／ 事例・トピック
【契約】プラン(梅/竹/松) / 契約開始日 / 更新日
```

---

## 5. GBP権限付与の具体手順（画面案内用）

顧客に画面共有で案内する：
1. Googleビジネスプロフィールを開く →「**ビジネス プロフィールの設定**」→「**ユーザーとアクセス権**」
2. 「**+ 追加**」→ こちらのメールアドレスを入力
3. 権限は「**管理者（Manager）**」以上（投稿・返信・写真UP・インサイト取得に必要）。可能なら Owner。
4. 招待メール → こちらが承諾

> **原則**：顧客を **Primary Owner のまま**にし、当社は **Manager or Owner**。顧客の資産である前提を崩さない。API代行にはこの権限付与が前提。

## 7. 時間短縮版（30分）とスケール時の注意
