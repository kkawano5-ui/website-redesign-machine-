# ココナラ「四柱推命×手相」自動鑑定サービス 運用計画

手のひらの写真と生年月日を受け取り、**四柱推命の命式を土台に手相を重ねた複合鑑定書**を自動生成して納品するサービス。バックエンド（鑑定文の生成）は全自動、ココナラ上の受付・納品だけ人が橋渡しする。

## 1. このサービスの正直な「自動化の範囲」

「うらっかわ全部自動」をやるうえで、最初に現実を押さえておく。

| 工程 | 自動化 | 補足 |
|---|---|---|
| 四柱推命の命式計算 | ✅ 完全自動 | `scripts/fortune/bazi.js`。決定論的・APIキー不要 |
| 手相画像の読み取り | ✅ 自動 | Claude Vision |
| 複合鑑定文の作成 | ✅ 自動 | Claude |
| 鑑定書のレイアウト | ✅ 自動 | Markdown＋納品用PDFを自動生成（`scripts/fortune/pdf.js`） |
| **ココナラでの受注・納品** | ⚠️ 手動 | ココナラは外部からの自動納品APIを公開していない。トーク画面で写真・生年月日を受け取り、生成結果をコピペ納品する |

➡️ つまり「**鑑定の中身は全自動、ココナラ操作だけ数分の手作業**」が現実的な落とし所。1件あたりの人間の作業は「依頼情報をJSONに転記 → `npm run fortune` → 出力をトークに貼る」で5分以内を目標にする。

## 2. プロダクト構成（コマンド）

```bash
# 命式だけ確認（APIキー不要・節入り境界の検証に便利）
npm run fortune -- data/fortune-inputs/sample.json --bazi-only

# 鑑定書を生成（要 ANTHROPIC_API_KEY）
npm run fortune -- data/fortune-inputs/sample.json
# → data/fortune-outputs/<日付>-<ニックネーム>.md

# エンジンの検算
npm run fortune:test
```

依頼1件 = `data/fortune-inputs/*.json` 1ファイル。フォーマットは `sample.json` 参照。

## 3. 商品設計（出品プラン例）

| プラン | 内容 | 価格目安 |
|---|---|---|
| ライト | 四柱推命のみ（生年月日だけ）／本質+全体運 | 1,000円 |
| スタンダード | 四柱推命×手相の複合鑑定書（1テーマ相談付き） | 2,500円 |
| プレミアム | 複合鑑定＋恋愛/仕事/金運の3テーマ＋開運アドバイス＋PDF納品 | 4,500円 |
| オプション | 追加相談1テーマ / 月運フォロー | +1,000円〜 |

> 価格はカテゴリ相場を見て調整。最初は実績（販売件数・評価）作りのため安め＋丁寧に。

## 4. 必要情報テンプレ（購入後トークで案内する定型文）

```
ご購入ありがとうございます！鑑定に下記を教えてください🙏
① お呼びする名前（ニックネーム可）
② 生年月日（西暦）：例 1992/7/23
③ 出生時刻：例 14:30（不明なら「不明」でOK。分かると精度が上がります）
④ 出生地（都道府県）：例 東京都
⑤ 手のひらの写真：利き手の手のひら全体。下の撮り方を参考にしてください
⑥ 相談したいこと：恋愛 / 仕事 / 金運 / 全体運 など
```

受け取った情報を `data/fortune-inputs/<id>.json` に転記して実行する。

## 5. 手のひら写真の撮り方ガイド（依頼者へ）

- 明るい場所で、手のひら全体が画面いっぱいに入るように
- 指を軽く広げ、シワ（線）がはっきり見えるピントで
- フラッシュの白飛びを避ける（自然光が理想）
- 利き手を基本に（両手送ってもらえるとなお良い）

## 6. 運用フロー（1件あたり）

1. 購入 → 定型文で必要情報を依頼
2. 情報が揃ったら `data/fortune-inputs/<id>.json` を作成（写真は同フォルダに保存）
3. `npm run fortune -- data/fortune-inputs/<id>.json` を実行
4. `data/fortune-outputs/*.md` を確認・微修正（人の目を1回入れる）
5. トーク画面にコピペ納品（プレミアムはPDF添付）
6. 評価をお願いする

## 7. 法務・表現の注意（重要）

- **エンターテインメント・自己理解目的**であることを出品文と鑑定書末尾に明記する。
- 病気・健康・寿命・余命の**診断や断定はしない**（医師法・医療類似行為と誤解される表現を避ける）。
- 「必ず」「絶対」など断定や不安を煽る表現を避ける（消費者保護・景表法的観点）。
- ココナラの占いカテゴリ規約（鑑定結果で重大な決断を縛らない等）に従う。
- 依頼者の写真・生年月日は個人情報。鑑定後は適切に管理し、無断で二次利用しない。

これらは `prompts/fortune-reading.md` のルールにも組み込み済み。

## 8. 集客の入口

- `fortune/index.html`：サービス紹介＋撮影ガイドのLP（Vercel等で公開し、SNSプロフィールやココナラ説明文からリンク）。
- まずはショート動画（手相→四柱推命の複合という切り口）でサンプル鑑定を見せ、プロフィールのココナラへ誘導するのが相性が良い。
