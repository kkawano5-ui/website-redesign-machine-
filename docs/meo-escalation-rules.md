# MEO エスカレーション・ルール（実装仕様 v1）

> **役割**：「何を自動で流し、何を人に上げるか」を機械判定するルール集。`scripts/meo/core/policy-check.js`（投稿の公開前ゲート）と口コミルーティングの実装仕様。
> **設計思想**：低リスクは自動公開、規制・トラブルリスクだけ人にエスカレーション。これで「45承認/日 → 本当に人が見るべき5〜7件/日」を実現する。
> **重要**：NGワード表は **v1 seed（たたき台）**。運用しながら追加する前提。薬機法・医療広告まわりは最終的に専門家レビューを推奨（ただし本チェッカーは"人へのエスカレーション"が最終防波堤なので、seedでも十分機能する）。

---

## 0. 全体アーキテクチャ

判定は2エンジン＋1プロファイルで構成する。

```
業種プロファイル（店舗ごと・オンボーディングで設定）
   industry: beauty | clinic | judo_seikotsu | realestate | legal | food | general
   → どの「規制ルールパック」を有効化するかを決める

エンジンA：口コミ・ルーティング（新着口コミ → 自動返信 or 人）
エンジンB：投稿ポリシーチェック（生成投稿 → 自動公開 or 修正 or 人）
```

### 重大度（severity）と処理
| severity | 意味 | 処理 |
|---|---|---|
| **HARD** | 規制違反・トラブルの恐れ | 🙋 **人にエスカレーション**（自動公開・自動返信しない） |
| **SOFT** | 軽微・機械的に直せる | 🔧 **自動修正**して公開（修正ログを残す）。直せなければ人へ |
| **CLEAN** | 問題なし | ✅ 自動で流す（投稿=公開／口コミ=自動返信） |

---

## エンジンA：口コミ・ルーティング

### A-1. 判定フロー
```
新着口コミ
  ├ ★1〜3 ?                          → 🙋 人（低評価は必ず人が返信）
  ├ 本文にエスカレーション・トリガー語あり? → 🙋 人（★4〜5でも上げる）
  ├ なりすまし/事実誤認の疑い?          → 🙋 人（＋Google削除申請フロー候補）
  ├ 新規店の最初の3件?                 → 🙋 人（信頼確立まで。以後自動へ）
  └ 上記すべて該当なし（★4〜5・クリーン）  → ✅ 自動返信文を生成→ポリシー通過なら自動返信
```

### A-2. エスカレーション・トリガー語（全業種共通・★不問で人へ）
| カテゴリ | 語の例（seed） |
|---|---|
| 健康被害・事故 | やけど / 火傷 / 腫れ / 痛み / 悪化 / かぶれ / アレルギー / 後遺症 / 怪我 / 事故 / 化膿 / 内出血 / 被害 |
| 法的・紛争 | 弁護士 / 訴え / 訴訟 / 消費者センター / 通報 / 返金 / 損害賠償 / 警察 / 詐欺 / 違法 |
| 強い不満・炎上 | 最悪 / 二度と / 最低 / ぼったくり / 騙され / 金返せ / 出禁 |
| 個人名指し・センシティブ | スタッフ実名を含む攻撃 / 差別的表現 / ハラスメント / 個人情報の暴露 |
| なりすまし・事実誤認（削除申請候補） | 行ってない / 来店してない / 別の店 / 知らない / 勘違い |

> ★4〜5でもトリガー語を含めば人へ回す（例：「★5だけど施術後に腫れた」は要対応）。**キーワード判定は星より優先**。

### A-3. 自動返信の条件（CLEANのみ）
すべて満たすときだけ自動返信：**①★4〜5 ②トリガー語なし ③生成した返信文がエンジンBの薬機法・誇大チェックを通過**。返信文は編集可能（公開後もGBPで修正できる）＝万一の事故は事後回収できる、を安全網とする。

## エンジンB：投稿ポリシーチェック（公開前ゲート）

生成した投稿は**公開前に必ずこのチェックを通す**。承認済みテーマの範囲＋CLEANなら自動公開。

### B-1. 全業種共通ルール（景表法・GBP投稿ポリシー）
| # | 検知 | 例 | severity | 処理 |
|---|---|---|---|---|
| U1 | 最上級・No.1（根拠なし） | 日本一 / No.1 / 業界最高 / 最先端 | HARD | 人へ（根拠・出典があれば可） |
| U2 | 断定・保証 | 必ず / 100% / 絶対 / 確実に | HARD | 人へ |
| U3 | 過度な値引き・煽り | 全品○%OFF / 激安 / 最安値 / 今だけ（乱用） | SOFT | 言い換え提案→ダメなら人 |
| U4 | 二重価格（根拠なし） | 通常価格○円→ / 定価の半額 | HARD | 人へ（根拠必要） |
| U5 | 本文に電話番号 | 090-… / 0120-… | SOFT | 自動除去（CTAボタンへ誘導） |
| U6 | 外部URL多用・不整合 | 本文に生URL連打 | SOFT | 整理 |
| U7 | 記号・絵文字スパム | 過剰な装飾 | SOFT | 自動整形 |
| U8 | 他社の名指し比較 | ○○より安い | HARD | 人へ |

### B-2. 業種別ルールパック（industryで有効化）

#### beauty（美容・エステ・ネイル・まつげ・化粧品）＝薬機法
医薬品的な効能効果の標榜が中心的NG。**痩身・アンチエイジングは特に厳格。**
| 検知カテゴリ | NG表現例（seed） | severity |
|---|---|---|
| 医薬品的効能 | 治る / 治療 / 改善する / 効く / 予防 / ○○に効果 | HARD |
| 美白・シミ断定 | シミが消える / シワがなくなる / 美白（範囲外の断定） | HARD |
| 痩身・脂肪 | 痩せる / 脂肪燃焼 / 脂肪分解 / サイズダウン保証 / デトックス / 毒素排出 | HARD |
| 若返り・細胞 | アンチエイジング / 若返り / 細胞が活性化 / ターンオーバー促進（断定） | HARD |
| 免疫・体質 | 免疫力アップ / 血行を根本改善 / 体質改善 | HARD |
| ✅ 言い換え可の範囲 | うるおいを与える / 肌を整える / ハリのある印象へ / リラックス / （メーキャップ効果で）明るい印象に | CLEAN |

#### clinic（歯科・美容クリニック・医療機関）＝医療広告ガイドライン
| 検知カテゴリ | NG例 | severity |
|---|---|---|
| 体験談・口コミ転載 | 患者の体験談を投稿に転載 | HARD |
| ビフォーアフター（説明不足） | 症例写真のみで詳細・リスク説明なし | HARD |
| 誇大・比較優良 | 最先端 / 絶対安全 / 苦痛なし / 日本初 | HARD |
| 自由診療の費用・リスク不記載 | 料金だけ／副作用・リスク明示なし | HARD |

#### judo_seikotsu（整骨・接骨・鍼灸・あん摩）＝あはき法・柔整法の広告制限
広告可能事項が限定列挙。適応症・効能の標榜は原則NG。
| 検知カテゴリ | NG例 | severity |
|---|---|---|
| 効能・適応症標榜 | 腰痛が治る / 肩こりに効く / ○○に効果的 / 治療 | HARD |
| 誇大 | 完全に改善 / 根本から治す | HARD |
| ✅ 可の範囲 | 施術内容の事実 / 営業時間 / 料金 / 施術者の資格 | CLEAN |

#### realestate（不動産）＝宅建業法・不動産表示規約
| 検知カテゴリ | NG例 | severity |
|---|---|---|
| 断定・最上級 | 完全 / 絶対 / 日本一 / 最高級 / 格安 / 掘り出し物 / 二度と出ない | HARD |
| おとり・不当表示 | 実在しない好条件物件 | HARD（要人判断） |
| 必要表示の欠落 | 取引態様・免許番号などの不記載 | SOFT→人 |

#### legal（弁護士・司法書士・税理士・行政書士）＝各会の業務広告規程・品位保持
| 検知カテゴリ | NG例 | severity |
|---|---|---|
| 成果保証 | 必ず勝てる / 100%還付 / 絶対に解決 | HARD |
| 誇大・比較 | 県内No.1（根拠なし）/ 業界最安 | HARD |
| 品位・秘密 | 依頼者が特定できる事例／品位を損なう表現 | HARD |

#### food / general（飲食・一般）
主に全業種共通ルール（U1〜U8）のみ。産地・最上級（景表法）に注意。**規制パックなし＝ほとんど自動公開に流れる**（＝エスカレーション最少）。

### B-3. 投稿の判定フロー
```
生成投稿
  ├ 承認済みテーマ外 / 新キャンペーン初回?  → 🙋 人
  ├ HARD ルールに一致?                     → 🙋 人（該当語＋修正候補を添えて）
  ├ SOFT ルールに一致?                     → 🔧 自動修正して公開（直せなければ人）
  └ すべてクリア                           → ✅ 自動公開（スケジュール）
```

---

## 3. エスカレーション時に人へ届くもの（UX）
承認待ちに積むときは、判断を1〜2秒で下せる情報をセットで：
- 対象（店舗名・投稿 or 口コミ）
- **何に引っかかったか**（ルールID・該当フレーズをハイライト）
- **修正候補**（投稿なら言い換え案、口コミなら返信ドラフト）
- ワンアクション：承認 / 修正して承認 / 却下

---

## 4. 想定エスカレーション率と運用の注意
- **コア3業種（美容・治療院・エステ）は薬機法・医療系が効く**ため、立ち上げ初期は投稿のHARD検知が多め（テンプレが規制を学習するまで）。**テンプレ／プロンプト側を"最初から薬機法セーフ"に作り込めば、数週間でエスカレーション率は大きく下がる**（弾かれない文章を最初から生成する）。
- 逆に **general/food はほぼ自動公開**に流れる。
- 口コミの人対応は「★1〜3＋トリガー語」なので、**全体の10〜20%程度**が目安（残り80%超は自動返信）。
- 数値は運用データで較正する（`data/meo-operations-ledger.csv` に「自動処理率／エスカレーション率」列を足すと可視化できる）。

## 5. 実装マッピング
| このルール | 実装先 |
|---|---|
| 業種プロファイル | `data/meo/stores/{id}.json` の `industry` フィールド |
| 全業種＋業種別NG表 | `scripts/meo/core/policy-check.js`（ルールをJSON/テーブルで外出しし、コード改修なしで語を追加できる形に） |
| 口コミ・ルーティング（★＋トリガー語） | `scripts/meo/reviews.js` の routing |
| 承認済みテーマ判定 | `store-config.js` の `approvedThemes` |
| エスカレーション・キュー | `scripts/meo/core/approval.js`（`data/meo/queue/` に出力） |
| 自動処理率の計測 | 台帳に列追加 |

> **設計原則**：NGワード・閾値は**コードにベタ書きしない**。`data/meo/policy/*.json` のような外部テーブルにして、**専門家レビューや運用の学びをコード改修なしで反映**できるようにする。
