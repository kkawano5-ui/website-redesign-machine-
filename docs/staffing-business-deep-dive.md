# 人材派遣業 業務プロセス 深掘りメモ

> 目的：AIの当て所を考える「前段」として、人材派遣業がそもそもどう回っているのかを業務レベルで理解する。
> （AI活用・カリキュラムは、この理解が固まってから別途重ねる。ここでは「派遣業とは何か」に集中する。）

---

## 1. 派遣業の根っこ＝「雇うのは自社、使うのは他社」という三者関係

派遣業のすべての業務と規制は、この一点から派生する。

- **派遣元（＝派遣会社・自社）** … スタッフと**雇用契約**を結ぶ。給与を払い、社会保険に入れ、雇用責任を負う。
- **派遣先（＝クライアント企業）** … スタッフに**指揮命令**して働かせる。が、雇ってはいない。
- **派遣スタッフ** … 雇われているのは派遣元、働く現場は派遣先。

ここが「請負」や「紹介」と決定的に違う。請負なら仕事の完成に責任を負うが、派遣は「人を送り、その人は派遣先の指示で働く」。紹介なら成約した時点で関係が切れるが、派遣は**就業が続く限り、雇用主としての義務（給与・社保・労務管理・安全配慮）がずっと自社に残る**。

> だから派遣業は「人を送って終わり」ではなく、**送ったあとの継続運用が本体**になる。
> 営業やマッチングは入口にすぎず、業務量の重心は「就業中のスタッフを何人、毎月正しく回し続けるか」にある。

---

## 2. 派遣にも種類がある（どの型かでビジネスの形が変わる）

ひとくちに派遣と言っても、雇用の持ち方で性格がかなり違う。

- **登録型派遣（有期雇用）** … いちばん一般的。スタッフは普段は「登録」状態で、仕事が決まった期間だけ雇用が発生する。短期〜長期まで柔軟。**仕事が途切れれば雇用も途切れる**ので、常に「登録プールの再活用」と「次の仕事への送り出し」が回り続ける。
- **常用型派遣（無期雇用）** … スタッフを期限なしで雇い続ける。**派遣先が決まっていない待機中も給与が発生する**ため、稼働率（=遊ばせない）が経営に直結する。同一部署3年の制限（後述）も対象外。
- **紹介予定派遣** … 最長6か月派遣したあと、本人と企業が合意すれば直接雇用に切り替える。派遣と紹介のハイブリッド。
- **日雇い派遣（スポット／単発）** … 原則**31日未満の派遣は法律で禁止**（一部の例外条件・業務のみ可）。「1日だけ」を気軽に回せるわけではない、という規制がここにかかる。

> 今回の先方は「スタッフが短期（ロープレイ）で稼働」「前払い（日払い）の仕組みがある」と話していた。
> つまり**登録型・短期回転が多い**業態とみられる。短期回転は、1人あたりの事務（契約・勤怠・給与・前払い）が**人数×回数ぶん発生する**＝事務負荷が構造的に重い、という特徴を持つ。

---

## 3. お金の流れ ＝ なぜ派遣が「薄利多売」なのか

派遣先が払う「派遣料金」は、まるごと派遣会社の儲けではない。中身はだいたいこうなっている（日本人材派遣協会の調査ベース）。

- **スタッフの賃金 … 約70%**
- **社会保険料（会社負担分・法定福利費）… 約10.9%**
- **有給休暇の費用 … 約4.2%**
  - ここまで「スタッフ関連費」で **約85%**
- **諸経費（営業・コーディネーターの人件費、オフィス・登録センター賃料、募集広告費、教育研修、相談窓口など）… 約13.7%**
- **営業利益 … 約1.2%**

つまり、**派遣料金の1.2%しか最終利益として残らない**。これが「薄利多売（農耕型）」の正体。

ここから読み取れる重要なこと：

1. **利益は「件数 × 運用効率」でしか伸びない。** 1件あたりが薄いので、回す件数を増やすか、運用コスト（諸経費13.7%の側）を削るかしかない。
2. **諸経費＝事務・営業・募集の人件費。ここがAIで効く可能性が最も高い領域。** 賃金や社保は法律で決まっていて削れないが、事務処理の手間は削れる。
3. **ミスが利益を直撃する。** 利益率1.2%だと、請求漏れや給与過払いが数件出ただけで一案件分の利益が吹き飛ぶ。だから先方が「ダブルチェックしたい」と言うのは、感情論ではなく**経済的に切実**。

## 4. 業務を担う2つの役割 ——「企業を見る人」と「人を見る人」

派遣会社の現場は、ざっくり2つの役割で回っている（会社によって兼務・呼称は違う）。

### 営業（派遣先＝企業側を担当）
- 新規開拓、既存クライアントへのヒアリング、求人ニーズの把握、求人票の作成、単価交渉、契約。
- 「現場を見て、どんな人を・何人・いつまで・いくらで」を企業から引き出すのが仕事。

### コーディネーター／CA（スタッフ＝人側を担当）
- スタッフ募集、求人原稿の作成・掲載、応募対応、**登録面談**、仕事紹介、職場見学のセッティング、雇用手続き、**就業後のフォロー**。
- 「この求人にこの人」を結びつけ、就業後も辞めないように支える。

典型的な1日のイメージ：朝にスケジュール確認 → 午前は企業から来た仕事をシステム登録して求人掲載 → 午後は応募者対応・企業への人材紹介・スキルシート作成・社内や営業との打合せ → 夜は初日就業を終えたスタッフに電話して様子を聞く。

> ポイントは、**この役割たちの一日が「転記・登録・作成・連絡」の反復で埋まっている**こと。
> 求人をシステムに入れ直す、スキルシートを作る、原稿を書く、契約や手続きの書類を作る、フォローの連絡を入れる——
> どれも「頭を使う判断」より「手を動かす定型」の比重が大きい。ここが諸経費13.7%の中身であり、削りどころ。

## 5. 契約まわりの「書類地獄」——派遣特有のいちばん面倒な所

派遣は、契約に入る前に**法律で決まった書類を全部そろえないと利用が認められない**。主な4点セット：

- **基本契約書**（派遣元⇄派遣先の大枠の取り決め。派遣元が作成）
- **個別契約書**（実際に人を受け入れるたびに、業務内容・期間・就業場所・指揮命令者・時間・賃金などを定める。**派遣ごとに発生**。派遣元が作成）
- **抵触日の通知**（派遣先 → 派遣元へ。下記の3年ルールの起点）
- **待遇に関する情報提供**（派遣先 → 派遣元へ。同一労働同一賃金のため）

さらにスタッフ向けには **就業条件明示書**（派遣法）と **労働条件通知書**（労基法）、社内管理として **派遣元管理台帳**、派遣先側にも **派遣先管理台帳** が要る。

何が大変かというと：

- **項目が多く、転記の塊。** 業務内容・期間・場所・指揮命令者・抵触日・賃金・時間外…を、契約・明示書・台帳に**同じ情報を何度も書き写す**。
- **派遣のたびに発生する。** 個別契約は1就業1通。短期回転が多いほど、書類の本数がそのまま増える。
- **下流すべての正本になる。** 先方が言った通り「**契約書さえ完璧なら、請求も給与も勤怠を当てて発行するだけ**」。逆に契約が雑だと、請求・給与の誤りに直結する。

> ここは「派遣特有の手間」が最も濃縮されている工程。理解の核心。

---

## 6. 就業後の継続運用 ——派遣業の「本体」

人を送ったあと、毎月これが回り続ける。短期回転だと月内に何度も発生する。

1. **勤怠の回収・承認** … スタッフの打刻を集め、派遣先に承認してもらい、締める。
2. **給与計算** … 勤怠を集計し、社会保険・雇用保険・税を控除して計算、明細発行、振込。
3. **前払い（日払い）** … 先方の例では「働いたその日に手当を払う」仕組みがあり、**毎朝メールが来た時点で人手で処理**している（「朝、絶対やらなきゃいけない」業務）。給与本体は別途 Workplace に月末集約。
4. **請求計算・請求書発行** … 同じ勤怠データを今度は派遣先への請求に変換して発行。
5. **スタッフフォロー・定着** … 就業中の相談対応、契約更新の管理。離職されると募集コストがまた発生するので、定着は利益に直結。

注目すべき構造：

- **勤怠データが「給与」と「請求」の両方の源泉。** 1つの勤怠から、スタッフへの支払いと企業への請求という**逆向きの2系統**が生まれる。だから「勤怠 → 給与」「勤怠 → 請求」の整合が崩れると、利益がそのまま漏れる。
- **毎月・全員ぶん必ず発生する純然たる定型。** 判断は少なく、正確性と締め切りがすべて。**自動化との相性が業務の中で最も高い領域**がここ。
- **前払いは「時間」がボトルネック。** 内容は単純でも「毎朝必ず」「朝までに」という時間制約が人を縛っている。

## 7. 規制が業務の形を決めている（コンプラ＝派遣の宿命）

派遣は許可制で、法律が業務のやり方そのものを規定している。主なもの：

- **3年ルール（抵触日）** … 同じ事業所・同じ人を、原則3年を超えて派遣できない（無期雇用は対象外）。**「いつ抵触日が来るか」を全就業で管理し続ける**必要がある。
- **日雇い派遣の原則禁止** … 31日未満は原則NG（例外あり）。短期ビジネスはこの線引きの中で設計される。
- **同一労働同一賃金（均等・均衡待遇）** … 派遣先の正社員と不合理な待遇差をつけてはいけない（2018年改正で強化）。だから派遣先からの「待遇情報の提供」が契約前提になっている。
- **マージン率の情報提供義務** … 派遣会社は自社のマージン率を公開しなければならない。
- **36協定** … 残業させるには、派遣元での協定締結・契約への明記など複数条件が要る。

> 規制は「面倒」であると同時に、**他社が真似しにくい参入障壁**でもある。
> 抵触日・同一労働同一賃金・36協定を正しく回す事務は専門性が高く、ここを仕組みで持っていること自体が価値になる
> （＝先方が狙う「他社に売れるオペレーション」の中核になりうる論点）。

---

## 8. まとめ：派遣業の「ペイン（手間と神経が溶ける所）」の地図

深掘りして見えてきた、業務負荷の濃い順のイメージ：

1. **契約書類の作成・転記**（個別契約が就業ごとに発生、項目が多い、下流の正本）
2. **毎月の給与・請求・勤怠の処理**（全員×毎月の純定型、ミスが利益を直撃）
3. **前払いの毎朝処理**（内容は単純だが時間制約がきつい）
4. **営業・募集まわりの作成業務**（求人原稿、スキルシート、提案、開拓のアプローチ）
5. **コンプラ管理**（抵触日・同一労働同一賃金・36協定の継続監視）
6. **スタッフフォロー・定着**（離職＝募集コスト再発）

そして全体を貫く事実が2つ：

- **利益は1.2%しか残らない** → 運用コスト削減とミス撲滅が、ほぼそのまま利益。
- **業務の大半が「転記・作成・計算・連絡」の反復定型** → 判断より手数が多い＝自動化の余地が大きい。

> 次の論点：この「ペインの地図」の上に、**どの工程にAIを当てると効果×実現性が高いか**を重ねる。
> （特に 1・2・3 は定型度が高く、契約という正本→給与/請求という派生、という流れが効くポイントになりそう。）
