## 1. 確定した運用モデル

### 1-1. 自動化の天井
- **生成・計測・レポートは全自動**。**公開（投稿・口コミ返信の実行）は「方針の事前承認＋例外時の個別承認」**で回す（＝1件ずつ全部承認はしない）。
- バックエンドは **GBP API を使う**前提（手動ログインでは自動公開できないため、API がこのモデルの前提条件）。

### 1-2. 役割分担（人 vs 自動）
| 領域 | 誰が | 備考 |
|---|---|---|
| **オンボーディング面談（30〜60分/社・オンライン）** | **人（本人）がやりたい** | ここは高付加価値。関係構築＋権限取得＋運用方針の合意＋自動化の同意を取る場 |
| 投稿の企画・文案・画像案・カレンダー生成 | 自動（Claude） | 承認済みテーマの範囲で |
| 投稿の公開 | 自動（API・スケジュール） | ポリシー通過分のみ。弾かれた分は人へ |
| 口コミ検知・返信文案 | 自動 | ★4〜5・NG無しは自動返信 |
| ネガ/要注意口コミの返信 | 人（例外承認） | ★1〜3・クレーム/医療/法務ワード |
| インサイト取得・月次レポート | 自動→自動送信 | 週1でまとめ目視のみ |

### 1-3. 「承認」の再定義（最重要）
- ❌ 旧：投稿1本・返信1件ごとに承認 → **120社で45承認/日＝非現実的**
- ✅ 新：**オンボーディングで"運用の憲法"を一度承認**（テーマ方針・自動返信の可否・エスカレーション基準）→ **以降は憲法内は自動、憲法を外れる例外だけ人に上がる**
- 効果：**45承認/日 → 本当に人が見るべき 5〜7件/日 ＋ 週1回30分のまとめ確認**

| コンテンツ | リスク | 扱い |

## 3. 工数モデル（竹プラン×120社・営業日20日/月）

| | 手動運用 | 自動化バックエンド後 |
|---|---|---|
| 1社あたり/月 | 約2.7時間 | 約0.87時間 |
| 120社合計/月 | 約324時間 → **2.5〜3人フル** | 約104時間 → **1人の半日** |
| 1日の人手 | 回すだけで破綻 | 承認5〜7件/日＋週1×30分 |

- **結論：自動化は"効率化"ではなく、事業計画（120社/人）を成立させる前提条件。** 未整備だと10〜20社で詰まる。
- **新しいボトルネックは本人のオンボーディング**：新規竹20〜24社/月なら月30時間弱（今は余裕）。フル稼働（48成約/月）になると本人のオンボーディングが律速→将来「面談の一部を人に渡す」判断が必要（今は good problem、後回し可）。

## 4. 提案書に反映すべきこと（別セッション向け指示）

`クラン株式会社様_MEO提案資料_最終`（Slides）を、`docs/meo-service-detail.md` を正として更新する。反映ポイント：

1. **「受け皿づくり」思想は維持**（順位保証しない語り口）。
2. **公開代行の位置づけを明確化（B確定）**：梅は「文案・提案まで／実行は顧客」、**竹は「投稿は公開まで代行（API・承認済みテーマは自動公開／立ち上げ期は手動）、口コミ返信は文案提示まで（顧客が返信）」**、**松は「投稿の高頻度＋口コミ返信も承認前提で代行＋競合分析・改善MTG・季節企画」**。→ 竹が主力として"投稿丸ごとお任せ"になる。
3. **オンボーディング面談の価値を1枚入れる**：30〜60分の面談で運用方針を一緒に決める＝"丸投げできる安心"を訴求。
4. **"正直にできないこと"のスライドを追加**：成果保証しない／自動公開はしない／虚偽・対価口コミはしない。これが 2026年4月のGoogle口コミポリシー厳格化（違反で全口コミ削除・BAN）から顧客資産を守る差別化になる、という文脈で。
5. **自動化×人の承認（human-in-the-loop）を安心材料として明示**：AIが下ごしらえ→人が薬機法・トーンを確認→公開。

## 4-2. 契約書に反映すべきこと
`MEO運用支援サービス契約書_最新` の「事前承認」の定義を微修正：
- 旧：投稿・返信ごとの事前承認
- 新：**運用方針の事前承認（オンボーディング時）＋ 例外時の個別承認**
- 併せて、自動化ツール利用・GBP権限付与・自動返信の範囲について同意条項を整理（既存の第11条 AIツール利用条項をベースに拡張）。

## 5. サービス思想の要点（3つの質問への答え・要約）

- **口コミ返信代行**：やれる。文案は全プラン。承認前提の代行返信は松のみ。`reviews API v4 updateReply` で技術的に可能（認証済み＋Owner/Manager権限）。公開スピードはGoogle審査依存で保証外。
- **投稿代行**：やれる。生成は全プラン全自動。**GBP公開代行は竹・松に搭載（B確定）**。竹＝承認済みテーマを自動公開／立ち上げ期は手動、松＝高頻度＋書面承認前提。`localPosts API`（2026年時点稼働）＋API承認が前提。
- **頻度スケール**：技術的にYES。生成・計測・レポは全自動、公開承認だけ人。承認が唯一のボトルネック＝1人あたり10〜20社（手動）／自動化＋例外処理型で大幅増。

## 6. 未決事項（次に詰める）

### 6-1. エスカレーション・ルールの具体化（作業項目①）→ ✅ **完了：`docs/meo-escalation-rules.md`**
- 口コミ：★1〜3＋トリガー語（健康被害/法的/炎上/なりすまし）で人へ、★4〜5クリーンは自動返信
- 投稿：全業種共通ルール（U1〜U8）＋業種別ルールパック（beauty=薬機法／clinic=医療広告GL／judo_seikotsu=あはき法・柔整法／realestate=宅建業法・表示規約／legal=品位保持）
- HARD＝人へ／SOFT＝自動修正／CLEAN＝自動公開の3段階
- → `scripts/meo/core/policy-check.js` の実装仕様。NGワードは `data/meo/policy/*.json` に外出しして専門家レビュー・運用の学びをコード改修なしで反映
- ⚠️ コア3業種（美容・治療院・エステ）はまさに薬機法・医療系が効く業種。テンプレを"最初から薬機法セーフ"に作れば初期以降エスカレーション率は大きく下がる

### 6-2. 公開代行をどのプランに入れるか → ✅ **B確定**
- **決定：B（竹にも投稿の公開代行を標準搭載）。** 竹29,800円は据え置き。
- 理由：バックエンドをAPI化する前提なら公開の追加コストが実質ゼロ。主力の竹（ターゲット＝忙しい美容/治療院/エステのオーナー）で"自分で投稿"の摩擦を消すことが、主力の解約防止に直結する。レポート/計測でどのみち全店のGBP APIアクセスは取るため、公開代行は"ついで"で成立。
- 松との差別化は維持：竹＝投稿は公開代行・口コミ返信は文案提示まで／松＝投稿高頻度＋口コミ返信も代行＋競合分析・改善MTG・季節企画。
- 注意：API承認は即日ではない（審査7〜10営業日〜最大6週間）。承認前・立ち上げ期は手動公開で回す（初期は店舗数が少ないため問題なし）。
