# 営業CRM 架電シート 作り替え（Apps Script）

対象スプレッドシート: **「50_営業CRM_727件」**
（https://docs.google.com/spreadsheets/d/10ZyTtYyqP246BmOAe0NcAKvE1YKZRG58DqHTK9Q41AU/edit ）

`CRM` タブを架電用にしっかり作り替え、`アポ獲得 / 電話アポ` を取った行を
`商談CRM` タブへ自動転送するための Google Apps Script です。

> **なぜスクリプト？**
> プルダウン（データ入力規則）や自動転送（onEdit トリガー）は、シートへ直接
> 流し込むことができず、Google Apps Script でしか設定できないためです。
> 下記の手順で1回実行すれば、すべて自動で構築されます。

---

## できること

`crm/CallTrackingSetup.gs` の `setupCallTracking` を実行すると、CRM タブが次のようになります。

### 1. 各架電回に「メモ」列を追加（各回 5 列構成）

| 1回目_架電担当 | 1回目_日付 | 1回目_時間 | 1回目_ステータス | **1回目_メモ** |
|---|---|---|---|---|

2回目・3回目も同じ 5 列構成（`_メモ` を追加）になります。

### 2. すべての架電列にプルダウン

| 列 | プルダウンの中身 |
|---|---|
| **架電担当**（各回） | 河野 / 高橋 / 北尾 |
| **ステータス**（各回） | 後日掛け直し / 資料送付 / お断り / アポ獲得 / 電話アポ / 留守 / 廃業 |
| **アポ形式** | オンライン / オフライン / 電話 |

### 3. アポ列の整備

`アポ日` の隣に **`アポ時間`** と **`アポ形式`** を追加します。
アポ獲得時はこの 3 つを埋めると、そのまま商談CRMへ反映されます。

### 4. ステータスの色付け（条件付き書式）

アポ獲得/電話アポ=緑、お断り=赤、留守=薄黄、廃業=グレー … など一目で分かるように。
アポなのに `アポ日` が未入力の行はオレンジで警告表示。

### 5. 「現在ステータス」を自動計算

- 最新回が **アポ獲得 / 電話アポ → 「アポ」**
- `商談日` 入力 → 「商談」 / `成約日` 入力 → 「成約」
- それ以外は最新回の結果を表示、未架電は「未着手」
- 各セルは手動上書きも可能

### 6. 「担当」（M 列）に最新の架電担当を自動表示

- 最新の架電回（**3回目 → 2回目 → 1回目**）で担当が入っている人を自動表示
- 架電担当を入力すると自動で更新（手動上書きも可能）

### 7. アポ → 商談CRM 自動転送（AF〜AJ もマネージ）

`アポ獲得 / 電話アポ` を選ぶと、その行が `商談CRM` タブへ **1 行転送** されます。
列は役割で色分けされます。

| 役割 | 色 | 列 | 挙動 |
|---|---|---|---|
| **自動同期** | グレー | CRM行 / place_id / 会社名 / 業種 / エリア / 電話 / マップURL / アポ種別 / 架電担当 / アポ日 / アポ時間 / 形式 | 転送のたびに CRM の最新値で上書き |
| **初期コピー** | 薄黄 | **サイト区分 / 既存website / 検出メール / うちのデモURL（AF〜AJ）** | 転送時に CRM から初期コピー。**以降は商談CRMで編集（マネージ）可** |
| **商談管理** | 白 | 商談ステータス（訪問予定→訪問済→商談中→成約/見送り）/ 次アクション日 / 商談メモ | 商談CRMで手入力 |

- 一度転送した行は商談が進んでも残り続けます（**自動削除しません**）。
- アポ日/時間/形式を後から入力すると、商談CRM 側（自動同期列）も自動で更新されます。
- 薄黄の「初期コピー」列は転送後に商談CRMで上書きしても、CRM 側から再同期で消されることはありません。

## 適用手順

1. 対象スプレッドシートを開く
2. メニュー **「拡張機能」→「Apps Script」**
3. `crm/CallTrackingSetup.gs` の中身を**すべてコピーして貼り付け**、保存（Ctrl/Cmd + S）
4. 関数の選択で **`setupCallTracking`** を選び、**▶ 実行**
   - 初回は Google アカウントの承認ダイアログ → **許可**
5. 右下に「✅ 完了」と出れば構築完了。以降は通常どおり架電入力するだけです。

> 再実行しても安全（冪等）です。列やプルダウンが既にあれば二重作成しません。

## 日々の運用

### IS（架電）
1. `N回目_架電担当`（プルダウン）→ `N回目_日付` / `N回目_時間`
2. `N回目_ステータス`（プルダウン）、必要に応じて `N回目_メモ`
3. **アポ獲得 / 電話アポ** のとき → `アポ日` / `アポ時間` / `アポ形式` を入力
4. → 自動で `商談CRM` に転送

### FS（商談・商談CRM タブ）
1. `商談CRM` で当日のアポを確認（会社名・業種・エリア・電話・マップURL）
2. `商談ステータス` を更新（訪問予定→訪問済→商談中→成約/見送り）
3. `次アクション日` / `商談メモ` を記入
4. 成約時は CRM タブの `商談日 / 成約日 / 商品 / 売上` も入力 → 現在ステータスが自動更新

## 補足・前提

- **通電の定義**: ステータスが「後日掛け直し / 資料送付 / お断り / アポ獲得 / 電話アポ」のいずれか
  （留守・廃業は通電に含めない）。
- **集計タブについて**: 旧ステータス語彙（通電/不在/再架電/訪問 等）を参照する数式が
  残っている場合、新語彙では 0 件になります。スクリプト内の **`auditSummaryFormulas`**
  を実行すると、見直しが必要なセルと数式を実行ログに一覧表示します
  （自動書き換えは行いません。集計を新語彙に合わせて組み直す作業はご希望あれば対応します）。
- このリポジトリには Apps Script ソースとドキュメントのみを格納しています。
  スプレッドシート本体はスクリプト実行時に更新されます。
